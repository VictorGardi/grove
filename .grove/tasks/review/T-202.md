## Description

Bridge the `opencode serve` SSE event stream into per-task Electron IPC channels. This is a pure main-process concern — subscribe to the SDK's event stream, filter to the relevant session, and push typed event objects to the renderer via `mainWindow.webContents.send`. No UI work here; that is T-203.

Depends on: T-200 (server), T-201 (session IDs).

## Implementation

### `src/main/ipc/opencodeEvents.ts` — SSE → IPC relay

**Subscription state:**

```ts
const subscriptions = new Map<string, { cancel: () => void }>();
```

**`opencodeEvents:subscribe({ taskId })`**

1. Look up `{ sessionId, serverUrl }` from session manager (T-201)
2. Get SDK client via `getClient(serverUrl)`
3. Call `client.event.subscribe(...)` (verify exact parameter shape against SDK types — may be `{ sessionID }` or `{ directory }`)
4. Iterate the async iterator:
   - **Filter**: only forward events where `event.properties.sessionID === sessionId` (prevents cross-task pollution when tasks share a directory subscription)
   - **Batch events** before forwarding: collect events over a 16 ms window and send as an array `mainWindow.webContents.send('opencode:event:${taskId}', events[])`. This avoids per-event structured-clone overhead for high-frequency `message.part.updated` bursts.
   - Forward ALL event types — the renderer decides what to render. Do not drop unknown types.
5. Store a cancel function in `subscriptions` map

**Reconnection (Grove-level, not SDK-level)**
- If the async iterator throws, check whether the SDK client has its own built-in retry (verify against SDK source). If the SDK retries internally, do not add a second retry loop — just surface the final error.
- If the SDK does NOT retry: wait `500ms * attempt`, retry up to 3 times, then send a synthetic `{ type: 'stream.error', message: '...' }` event on the taskId channel.

**`opencodeEvents:unsubscribe({ taskId })`**
- Call `cancel()` from subscriptions map
- Delete from map

### `src/main/ipc/index.ts`

Call `registerOpencodeEventsHandlers()` inside `registerIpcHandlers()`.

### `src/preload/index.ts` — push-subscription pattern

Follow the existing `api.pty.onData` pattern — expose a function that registers a listener and returns an unsubscribe function:

```ts
api.opencodeEvents = {
  // Returns an unsubscribe function. Call it from useEffect cleanup.
  onEvent(taskId: string, callback: (events: OpenCodeEvent[]) => void): () => void {
    const channel = `opencode:event:${taskId}`;
    const listener = (_: IpcRendererEvent, events: OpenCodeEvent[]) => callback(events);
    ipcRenderer.on(channel, listener);
    return () => ipcRenderer.removeListener(channel, listener);
  }
};
```

The preload layer enforces the cleanup contract — the caller (renderer component) must call the returned function in `useEffect` cleanup. This prevents listener accumulation on task restart or component remount.

### Event types to handle (verify names against actual SDK exports)

The relay forwards everything. The following are expected in the UI (T-203) and must not be dropped:
- `message.part.updated` — streaming text/tool delta
- `session.status` — includes `{ type: "idle" | "busy" | "retry" }`
- `session.error` — structured error
- `tool.use` — tool call initiated
- `tool.result` — tool call result
- `permission.asked` — agent is requesting user permission (must surface to UI, not auto-dropped)

**Important**: `session.status` has THREE states, not two. `retry` means the agent is retrying after a provider error and must be forwarded.

**Important**: `permission.asked` must NOT be silently dropped. If dropped, the agent will stall waiting for a response it never receives. Until T-204 adds a permission dialog, forward this event to the renderer and render it as a visible warning.

## Definition of Done

- [x] `opencodeEvents:subscribe` subscribes to the SSE stream for a given taskId/sessionId
- [x] Events are filtered by `sessionID` before forwarding — tasks sharing a worktree directory do not receive each other's events
- [x] Events are batched over a 16 ms window before `webContents.send` (avoids per-event structured-clone overhead)
- [x] ALL event types are forwarded — the relay does not drop unknown types
- [x] `session.status { type: "retry" }` is forwarded (not treated as idle or busy)
- [x] `permission.asked` events are forwarded to the renderer (not silently dropped)
- [x] If the SDK has built-in SSE retry, Grove does not add a second retry loop
- [x] If the SDK does not retry: Grove retries up to 3 times with exponential backoff (500ms, 1000ms, 2000ms), then sends a synthetic `stream.error` event
- [x] `opencodeEvents:unsubscribe` cancels the stream and removes the subscription
- [x] Preload bridge exposes `api.opencodeEvents.onEvent(taskId, cb)` returning an unsubscribe function
- [x] IPC handlers registered in `src/main/ipc/index.ts`
- [x] No changes to the PTY/tmux push pattern for existing agents
