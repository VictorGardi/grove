## Description

T-198 simplified the preamble content but kept the intermediate file layer. The current flow builds a 4-line preamble, writes it to `.grove/context/context-{session}.md` via `taskterm:writecontext`, then injects `"Please read {contextPath} for your task context and instructions."` into the terminal. The preamble is short enough to inject directly — the file write is unnecessary indirection. The `.grove/context/` directory and its files should stop being created.

## Definition of Done

- [x] `injectContext.ts` injects the 4-line preamble + task content as the direct PTY message — no `writeContext` call, no context file path
- [x] No new `.grove/context/` files are created during injection
- [x] `taskterm:writecontext` IPC handler removed from `src/main/ipc/taskTerminal.ts`
- [x] `writeContext` and `cleanContext` stubs removed from preload bridge (`src/preload/index.ts` and `src/preload/index.d.ts`)
- [x] Context file cleanup block removed from `taskterm:kill` handler
- [x] Dead imports (`CONTEXT_DIR`, and any of `fs`/`os`/`path` no longer used) removed from `taskTerminal.ts`
- [x] `taskContent` is now extracted from params and included in the preamble so the agent receives the full task description
