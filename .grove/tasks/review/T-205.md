## Description

When a user selects "claude" in the launch modal and starts execution, opencode starts instead. There are three compounding root causes:

1. **`src/main/tasks.ts` whitelist bug (critical):** The frontmatter deserializer only accepts `"opencode"` and `"copilot"` as valid values for `execSessionAgent` and `planSessionAgent` — `"claude"` is coerced to `null` on every file-system round-trip. So `task.execSessionAgent` is always `null` for claude users, regardless of what was written to frontmatter.

2. **Wrong field in `TaskDetailPanel.tsx` (~line 674):** `effectiveAgent` is computed from `task.agent` (a legacy field that is never written during execution setup) instead of `task.execSessionAgent`. Even after fixing the whitelist, this would still cause the wrong agent prop to reach `TaskTerminal`.

3. **Race condition — double session creation:** When the task detail panel is open as the user initiates execution, `TaskTerminal` remounts (key changes from plan→exec) with no session name and auto-starts (300ms timer). Concurrently, `executionActions.ts` also calls `window.api.taskterm.create`. Both use the same deterministic tmux session name and PTY ID. The second call kills the first session, leaving the `TaskTerminal` PTY attached to a dead session → "Session ended" flicker.

**Fix (four changes):**

1. `src/main/tasks.ts` — add `"claude"` to the whitelist for both `execSessionAgent` and `planSessionAgent` deserialization.

2. `src/renderer/src/components/TaskDetail/TaskDetailPanel.tsx` — fix `effectiveAgent` to use `task.execSessionAgent` for doing/review tasks and `task.planSessionAgent` for backlog tasks (with workspace defaults as fallback), instead of the legacy `task.agent` field.

3. `src/renderer/src/actions/executionActions.ts` — remove the `window.api.taskterm.create` and `injectExecutionContext` calls (and the orphaned `isRunning` guard that guarded them). `TaskTerminal` becomes the sole session creator for exec mode. Note: `executionActions.ts` still handles worktree setup, frontmatter writes (agent, model, useWorktree), and task status move.

4. `src/renderer/src/components/TaskDetail/TaskTerminal.tsx` — replace the weaker `sendInitialContext` helper with a call to the shared `injectExecutionContext` from `utils/injectContext.ts`. This restores retry logic, marker-echo acknowledgement, and `terminalExecContextSent` persistence that would otherwise be lost by removing the call from `executionActions.ts`.
