## Description

Spin up and manage a single `opencode serve` process for the Grove app instance. One server handles all worktrees — the `directory` parameter is scoped per API call, not per server. Expose the server's base URL to the renderer via preload so subsequent tasks can create SDK clients.

This is the foundation task. Nothing in T-201–T-204 works without it.

## Implementation

### `src/main/opencodeServerManager.ts` — singleton server manager

- `ensureServer()`:
  1. Check if a server is already tracked in module-level state; if so, verify it is still alive (HTTP GET `/health` with 500 ms timeout)
  2. If alive, return `{ url }`
  3. If not alive (or never started): spawn `opencode serve` with `{ port: 0 }` flag so the OS assigns a free port; capture the assigned port/URL from the first line of stdout (opencode prints the listening URL)
  4. If port 4096 is already in use by an existing `opencode serve` (not Grove-managed): do a GET `/health`, confirm the response body includes an expected marker, then adopt it and store `{ url: 'http://localhost:4096' }` without spawning a new process
  5. Pass `doom_loop: allow` via the `OPENCODE_CONFIG_CONTENT` env var on spawn (reuse/adapt `writeOpencodeConfig` from `src/main/opencodeConfig.ts`)

- `killServer()`: send SIGTERM to tracked process; wait up to 2 s then SIGKILL

- `getServerUrl()`: returns tracked URL or `null`

- **Crash restart**: attach `process.on('exit', ...)` to the spawned child; on unexpected exit, wait 2 s and call `ensureServer()` again; max 3 auto-restarts before storing an error state and forwarding it to renderer on next `ensure` call

- **App quit cleanup**: `app.on('will-quit', killServer)` in `src/main/index.ts`

### `src/main/ipc/opencodeServer.ts` — IPC handlers

- `opencodeServer:ensure` → calls `ensureServer()`, returns `{ url: string }` or `{ error: string }`
- `opencodeServer:kill` → calls `killServer()`
- `opencodeServer:status` → returns `{ running: boolean; url: string | null; pid: number | null }`

### `src/preload/index.ts` — preload bridge additions

Add `api.opencodeServer` namespace following the exact pattern of existing namespaces:

```
api.opencodeServer.ensure(): Promise<{ url: string } | { error: string }>
api.opencodeServer.kill(): Promise<void>
api.opencodeServer.status(): Promise<{ running: boolean; url: string | null; pid: number | null }>
```

### `src/main/ipc/index.ts`

Call `registerOpencodeServerHandlers()` inside `registerIpcHandlers()`.
