## Problem

The "Agent exited without output. The session may have expired — try New session." warning fires as a **false positive** while the tmux session and opencode session are still alive. The agent responds later in the same session. This is a recurring UX issue.

## Root causes identified

### Root cause A — `done` chunk triggers `startAgentMessage` in App.tsx

The condition that lazily creates an agent bubble included `chunk.type === "done"`. A `done` chunk should only close the current bubble, never open a new empty one.

### Root cause B — `applyChunk(done)` unconditionally sets `isRunning: false` (usePlanStore.ts)

Both intermediate `done` chunks (multi-turn boundary during replay) and the final `done` chunk went through the same handler. An intermediate `done` must not reset `isRunning`.

### Root cause C — Peek logic used `tailer.offset` (end of buffer) instead of per-line offset

If a sentinel and subsequent content were in the same 65536-byte read buffer, `tailer.offset` overshoot past both, making the peek incorrectly return EOF → misidentified intermediate sentinel as final.

### Root cause D (actual cause of "Is opencode on PATH?" flash) — stale `activeKeys` in `TmuxPlanRunner`

`TmuxPlanRunner.start()` calls `cancel(runKey)` at the start of every new send to abort any in-flight run. `cancel` emits a synthetic `done` with exit code **1**. However, `activeKeys` was never cleaned up when a run completed normally via the grove_exit sentinel — only when `cancel()` was called. So every follow-up send found the previous run in `activeKeys`, fired `cancel`, emitted `done{code:1}` into the chunk stream, and the renderer briefly showed `isRunning: false` + `lastExitCode: 1` → "Is opencode installed and on PATH?" warning for ~1 second.

## Fixes applied

1. **App.tsx**: Removed `"done"` from the `startAgentMessage`-trigger condition.
2. **App.tsx**: `replay_done` now also calls `setRunning(false)` (since intermediate `done`s during replay no longer set it).
3. **usePlanStore.ts**: `done` handler during replay only closes the bubble; does NOT set `isRunning: false`.
4. **tmuxSupervisor.ts**: Peek now uses per-line `afterLineOffset` tracked in `drain()`. Added `lineBufferOffset` field to `LogTailer`.
5. **planManager.ts**: `TmuxPlanRunner.start()` wraps `onComplete` to delete `runKey` from `activeKeys` and `runCallbacks` when a run completes normally. Prevents spurious `cancel` + synthetic `done{code:1}` on follow-up sends.
