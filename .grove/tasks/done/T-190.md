## Description

When a user selects an agent in the dropdown, the model dropdown shows models for the **wrong agent**. The root cause is in `src/renderer/src/components/Settings/AgentModelForm.tsx`.

### Root Cause

When `planningAgent` or `executionAgent` changes:
1. The cache key `${workspacePath}:${planningAgent}` updates correctly
2. `ensureModels()` is called to fetch models for the new agent
3. **BUT** the local state `planningModels`/`executionModels` is NOT cleared
4. Old models remain displayed until new cache entry loads

This causes a race condition where models from agent A briefly appear in the dropdown after switching to agent B.

### Affected Files

- **Primary:** `src/renderer/src/components/Settings/AgentModelForm.tsx` — model dropdown shows wrong models on agent switch
- **Secondary:** `src/renderer/src/components/TaskDetail/TaskTerminal.tsx` — two bugs:
  1. Missing "claude" option in agent dropdown (line 746-747)
  2. May have same model-display bug as AgentModelForm (lines 335-341)

### Proposed Fix

**1. TaskTerminal.tsx — Add missing "claude" option (line 746-747):**

```tsx
<select value={selectedAgent} onChange={(e) => setSelectedAgent(...)}>
  <option value="opencode">opencode</option>
  <option value="copilot">copilot</option>
  <option value="claude">claude</option>  {/* ADD THIS */}
</select>
```

**2. AgentModelForm.tsx — Clear models on agent change:**

```typescript
// Clear planning models when agent changes
useEffect(() => {
  setPlanningModels([]);
  setPlanningModelsLoading(true);
}, [planningAgent]);

// Clear execution models when agent changes
useEffect(() => {
  setExecutionModels([]);
  setExecutionModelsLoading(true);
}, [executionAgent]);
```
