## Description

Add keyboard arrow-key navigation to the kanban board so users can move through task cards and open them without using a mouse.

### Implementation plan

1. **Add `focusedTaskId` to `useBoardStore`** — a transient focus state separate from `selectedTaskId` (which controls the detail panel).

2. **Add arrow key handlers in `useKeyboardShortcuts.ts`** — only active when `selectedTaskId === null` (detail panel closed) and not in board search input:
   - **Up/Down**: Navigate within the current column
   - **Left/Right**: Jump to the same relative index in the adjacent non-empty column (skip empty columns)
   - **Home/End**: Jump to first/last task in the current column
   - **Enter**: Call `setSelectedTask(focusedTaskId)` to open the detail panel

3. **Wrap behavior**: Down from the last card in DONE wraps to the first card in BACKLOG. Up from the first card in BACKLOG wraps to the last card in DONE.

4. **Visual indicator**: Add a `.cardFocused` CSS class to `TaskCard.module.css` with the same styling as `.cardSelected`. TaskCard reads both `focusedTaskId` from `useBoardStore` and `selectedTaskId` from `useDataStore` to apply the combined class.

5. **Focus lifecycle**:
   - **Initial focus**: When pressing an arrow key with no `focusedTaskId` set, default to the first task in the BACKLOG column
   - **Cleared on**: workspace switch, Escape key, clicking a card
   - **Search filtering**: When search is active, only navigate visible (filtered) cards. If the currently focused task is filtered out, clear focus.

6. **B/D/R/F shortcuts**: Unchanged — they continue to operate on `selectedTaskId` only.

### Files to modify

- `src/renderer/src/stores/useBoardStore.ts` — add `focusedTaskId`, `setFocusedTask`, `clearFocusedTask`
- `src/renderer/src/hooks/useKeyboardShortcuts.ts` — add arrow key, Home/End, Enter handlers
- `src/renderer/src/components/Board/TaskCard.tsx` — read `focusedTaskId` and apply `.cardFocused` class
- `src/renderer/src/components/Board/TaskCard.module.css` — add `.cardFocused` class (same as `.cardSelected`)
- `src/renderer/src/stores/useDataStore.ts` — clear `focusedTaskId` in `clearSelectedTask` and `clear`
- `src/renderer/src/stores/useWorkspaceStore.ts` — clear `focusedTaskId` on workspace switch
