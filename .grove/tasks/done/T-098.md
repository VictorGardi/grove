## Description

Simplify the UI by removing the global model badge above chat and unifying agent selection with a slash command.

### Current State

- Global model badge in header shows `model || "default"` - but is confusing/redundant since per-message badges show actual model used
- Agent dropdown in header is disabled after session starts
- Only `/model` slash command exists, no `/agent` command
- `handleAgentChange()` blocks mid-session agent changes (inconsistent with `/model`)

### Changes

1. **Remove global model badge** from header (lines 1080-1085 in PlanChat.tsx)
   - Rationale: Per-message model badges are more useful - they show actual model used per message

2. **Add `/agent` slash command** in chat input
   - Works mid-session (like `/model` does)
   - Available agents: "opencode", "copilot"

3. **Remove header agent dropdown** (`<select>` at lines 1071-1079)
   - Rationale: Unify to single entry point via `/agent` command

4. **Fix `handleAgentChange()`** to allow mid-session changes
   - Remove `hasSession` check (keep only `isRunning` block)
   - Align behavior with `/model` which allows mid-session changes

5. **Update placeholder text** to mention `/agent`
