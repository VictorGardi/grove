## Description

When a task is dragged to "doing", the Board.tsx creates a terminal session and injects execution context (via `injectExecutionContext`). It then sets `terminalExecContextSent: true` in the task frontmatter.

However, when the user opens the task detail panel and TaskTerminal mounts, it takes the reconnect path (since `initialSessionName` is set from frontmatter). The component checks `task.terminalExecContextSent` to decide whether to re-inject context, but this value comes from the Zustand store (`useDataStore`), which is stale — it was populated on initial load and hasn't refreshed the frontmatter value.

This causes TaskTerminal to re-inject the context. After injection, the agent becomes idle, which triggers the agent to respond, which makes it idle again — and the cycle repeats.

### Root Cause

1. Board.tsx creates session + injects context + sets `terminalExecContextSent: true` in frontmatter
2. User opens task panel → TaskTerminal mounts
3. TaskTerminal sees `initialSessionName` set → takes reconnect path
4. TaskTerminal checks `task.terminalExecContextSent` (from stale Zustand store) → sees `false`
5. TaskTerminal re-injects context via `injectAfterReconnect`
6. Agent becomes idle → re-injects → triggers agent → becomes idle → repeat

### Fix

1. **Read fresh frontmatter on reconnect**: In TaskTerminal's mount effect, before deciding to inject, read the current frontmatter directly (via `window.api.tasks.readRaw` or a new lightweight IPC) to get the actual `terminalExecContextSent` value instead of relying on the potentially stale task prop from the store.

2. **Consider session name matching**: Also verify that the `terminalExecSession` in frontmatter matches the session we're reconnecting to (to handle edge cases where session was killed and recreated with same name).

## Definition of Done

- [x] TaskTerminal checks fresh frontmatter value (not stale store) when deciding to inject on reconnect
- [x] No duplicate context injection when reconnecting to an existing session that already received context
- [x] Agent doesn't receive repeated "Please read..." prompts after becoming idle
- [x] Edge case: If Board injection fails/times out, TaskTerminal can still inject on mount (not blocked by stale `terminalExecContextSent: true`)
- [x] Edge case: Handle case where session was killed and new session created with same name
