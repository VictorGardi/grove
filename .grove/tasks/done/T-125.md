## Description

Grove writes session context files to `/tmp/grove-context-{session}.md` so agents (opencode, copilot) can read task instructions. When an agent starts a session from a repo worktree and tries to read these temp files, it triggers permission prompts.

Currently, Grove writes `opencode.json` with `external_directory: "allow"` which permits ALL external directories — too permissive.

**Proposed solution:** Write context files to `~/.grove/` (global location) instead of `/tmp`, and configure opencode.json to allow only that specific path. This is more secure (only grants access to Grove's own directory) and works across any repo.

### Implementation details

1. **Write context files to `~/.grove/`** instead of `os.tmpdir()`
   - Path: `~/.grove/context-{sessionName}.md`
   - Ensure directory exists before writing

2. **Configure opencode.json with path-specific permission**
   - Current: `external_directory: "allow"` ( blanket allow)
   - New: `external_directory: { "~/.grove/**": "allow" }` (specific path)

3. **For copilot CLI:**
   - Use `--add-dir=~/.grove` flag when invoking `gh copilot suggest`
   - Or set `COPILOT_ALLOW_ALL=true` environment variable (but this is too permissive)
   - Better: use `--add-dir` to allow only the specific directory
