### Migration Strategy

**CRITICAL: Auto-migration on startup**

1. **Detection**: On workspace load in `src/main/ipc/workspace.ts`, check if migration needed:
   - If `.tasks/` exists at root but `.grove/tasks/` doesn't â†’ trigger migration
   - If both exist â†’ use `.grove/tasks/` (new) but log warning about stale old directory
   - If only `.grove/tasks/` exists â†’ use new (normal, no migration needed)
   - If neither exists â†’ create fresh `.grove/tasks/` structure

2. **Migration process** (in `src/main/migration.ts` - new file):

   ```
   function migrateWorkspace(workspacePath: string):
     1. Create .grove/tasks/, .grove/tasks/{all status dirs}
     2. Create .grove/decisions/ if .decisions/ exists
     3. Copy all files from .tasks/ -> .grove/tasks/{status}/
     4. Copy all files from .decisions/ -> .grove/decisions/
     5. Verify all files copied correctly (checksum/count)
     6. If verification passes, delete old .tasks/ and .decisions/
     7. Write migration flag to .grove/config.json: { migratedAt: "2026-04-18" }
   ```

3. **Backward compatibility during transition**:
   - CLI workspace detection checks BOTH `.tasks/` AND `.grove/tasks/`
   - If old path found but new doesn't exist â†’ run migration first
   - If both exist â†’ use new as source of truth, warn about stale old

4. **Gitignore**: Add `.grove/context/` to repo's `.gitignore`:
   ```
   # Grove
   .grove/context/
   ```

### Edge Cases

| Edge Case                         | Handling                                                               |
| --------------------------------- | ---------------------------------------------------------------------- |
| **Git worktrees**                 | Each worktree gets its own `.grove/` directory at worktree root        |
| **Partial migration**             | Detect both old+new, warn user, use new as source of truth             |
| **Migration fails mid-way**       | Leave old intact, report error, don't delete old until verified        |
| **CLI-only mode**                 | Migration runs on any workspace access (main or CLI)                   |
| **Permission errors**             | Fail gracefully with clear error: "Failed to create .grove/ directory" |
| **Large repos**                   | Migration runs async, log progress                                     |
| **Old context files in ~/.grove** | Leave for recovery, new sessions use `.grove/context/`                 |
| **Path normalization**            | Use `path.join` consistently for cross-platform (Windows handling)     |

## Context for agent

**Original problem:** Context files stored in `~/.grove/` require special agent permissions.

**Investigated alternatives:**

- **Keep ~/.grove for context only**: Requires special agent permissions, inconsistent with existing `.grove/config.json` pattern
- **Temp dir**: OS cleanup but harder debugging, access issues
- **Repo-local .grove/**: Agents have natural access via repo, consistent with existing pattern

**Extended scope:** During investigation found that `.tasks/` and `.decisions/` at repo root are also not ideal - they're Grove internal data that should be alongside `.grove/config.json`.

**Selected:** Repo-local `.grove/` containing tasks, decisions, and context.
