## Description

When the external terminal panel (Cmd+J) is hidden, `TerminalPanel.tsx:70-75` correctly moves all visible tabs into a `hiddenTabs` Map (PTYs stay alive). However, when the panel is re-opened, nothing restores those tabs back. The auto-create effect in `App.tsx:202-220` checks only the live `tabs` array — finds it empty — and spawns a brand-new PTY session, discarding the preserved one.

**Root cause:** `restoreTab` exists in `useTerminalStore` but is never called on panel re-open. The hide and auto-create logic live in different components (`TerminalPanel.tsx` vs `App.tsx`), so a naïve fix risks a cross-component ordering race.

**Fix approach:** Consolidate restore + auto-create into the single effect in `App.tsx:202-220`:
1. When `terminalPanelOpen` is true, check `hiddenTabs` (filtered by `activeWorkspacePath`) for tabs to restore.
2. If hidden tabs exist → call `restoreTab` for each, preserving original insertion order.
3. Only if no live tabs AND no hidden tabs for the workspace → create a new free tab.

The hide-to-`hiddenTabs` logic in `TerminalPanel.tsx:70-75` remains unchanged.

## Definition of Done

- [x] Toggling the terminal panel closed (Cmd+J) and re-opening it in the same app session restores the same terminal session(s) — command history visible, working directory preserved
- [x] Multiple open terminal tabs are all preserved with their original order when the panel is hidden and restored
- [x] Opening the terminal for the first time in a session (no hidden tabs anywhere for the workspace) still creates a new free tab
- [x] Terminal sessions are NOT persisted across app restarts
- [x] Switching workspaces while the panel is open continues to show only the active workspace's tabs
- [x] Switching workspaces while the panel is hidden, then re-opening it, restores the new workspace's hidden tabs without creating new PTYs and without destroying the old workspace's hidden tabs
