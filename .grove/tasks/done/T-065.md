## Description

The task card status indicator frequently shows "Waiting for input" when the agent is actually still running in the background (editing files, processing, etc.).

**Root cause**: The `isRunning` flag in `usePlanStore` is purely event-driven from streaming chunks, not derived from the actual tmux process state. This creates multiple failure modes:

1. **Store reset on component remount**: `isRunning` defaults to `false` on session init (`usePlanStore.ts:114`). If the PlanChat component remounts mid-run, the flag resets to false.

2. **Reconnect race conditions**: The reconnect effect (`PlanChat.tsx:355-419`) attempts to restore `isRunning: true`, but there's a timing gap where it shows as false.

3. **No actual process verification**: The UI doesn't verify the tmux session is actually running — it relies entirely on in-memory state that can get out of sync.

**Key files**:

- `src/renderer/src/components/Board/TaskCard.tsx:23-31` — `isAgentRunning` logic
- `src/renderer/src/components/Board/TaskCard.tsx:36-41` — `isExecuteWaiting` logic
- `src/renderer/src/stores/usePlanStore.ts:114` — `isRunning: false` default
- `src/renderer/src/components/TaskDetail/PlanChat.tsx:324-333` — reset stale isRunning on mount
- `src/renderer/src/components/TaskDetail/PlanChat.tsx:355-419` — reconnect effect

**Proposed fix approaches**:

1. Add a fallback that checks `tmux has-session` when `isRunning` is false but session ID exists
2. Persist `isRunning` state alongside the session ID in frontmatter
3. Use the `sessionStatus` field (which has "running"/"paused"/"idle"/"reconnecting" states) instead of `isRunning` for the UI indicator

## Definition of Done

- [x] "Waiting for input" only shows when the agent has actually finished (tmux session no longer exists or exit code received)
- [x] "Agent running" shows whenever the tmux session is alive, even if the app was restarted
- [x] No false "waiting for input" states during normal operation (component remounts, navigation, etc.)
- [ ] Manual testing with long-running tasks to verify status stays accurate
- [x] Edge case: app restart while agent is running correctly shows "agent running" on reopen
