## Description

Replace the per-task model dropdown on `TaskCard` with a **LaunchModal** that opens when a task is dragged to the "doing" column. Instead of surfacing execution configuration inline on every card, the modal surfaces at the moment of intent — just before execution starts.

**Modal contents:**

- Task title (display only)
- Agent selector (`opencode` / `copilot`), pre-filled from 3-level fallback: `task.execSessionAgent → defaults.defaultExecutionAgent → "opencode"`
- Model dropdown, pre-filled from 3-level fallback: `task.execModel → defaults.defaultExecutionModel → null ("default")`, populated lazily from `modelsCache`
- Use worktree toggle (checkbox/switch), pre-filled from `task.useWorktree ?? true`
- **"Execute"** (primary) and **"Cancel"** (secondary) buttons

**Behaviour:**

- On Cancel → modal closes, task stays in its original column, no IPC calls, no file moves, no side effects
- On Execute → `handleDragToDoing` runs with the user-selected values; agent, model, and useWorktree are persisted to frontmatter
- Changing the agent in the modal triggers a new `ensureModels` fetch for the new agent; model dropdown shows a loading state while `modelsCache` is `null` and resets the selected model if it is not present in the new agent's model list

**Removals:**

- Inline `execModel` `<select>` removed from `TaskCard` (both backlog and doing columns)
- Inline `useWorktree` toggle removed from `TaskCard` (moved entirely into the LaunchModal; there is no post-launch UI need to change it since the worktree is already created)
