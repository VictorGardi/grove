## Description

The `TaskTerminal` component has a start-screen textarea where users type their initial request before launching an agent session. Currently this textarea has no file-mention support. `PlanChat.tsx` â€” the older headless-agent chat UI â€” had `@`-triggered file autocomplete but is **no longer rendered anywhere** (`TaskDetailPanel.tsx` uses `TaskTerminal` exclusively). `ToolUseBlock.tsx`, `TodoListBlock.tsx`, and `PlanChat.module.css` are also dead, used only by `PlanChat`.

This task has two parts:

1. **Add `@path` autocomplete to `TaskTerminal`'s start-screen textarea.** When the user types `@` followed by any characters, a dropdown appears with fuzzy-searched workspace files (via `useFileStore` + Fuse.js, which is already a project dependency). Selecting a file inserts `@full/path` at the cursor. The agent receives the path as inline text in the message and is responsible for reading it â€” no file content is pre-loaded.

2. **Remove the dead code cluster**: `PlanChat.tsx` (1,569 lines), `ToolUseBlock.tsx`, `TodoListBlock.tsx`, and `PlanChat.module.css`. The `buildFirstPlanMessage`/`buildFirstExecutionMessage` re-exports in `PlanChat.tsx` are safe to delete â€” `TaskTerminal.tsx` and `Board.tsx` both already import these directly from `utils/planPrompts.ts`. The `usePlanStore`, `agentRunner.ts`, and the `plan` IPC layer are **not** dead and must be left intact (used by Board drag-to-doing auto-execute).

## Definition of Done

### @ file autocomplete in TaskTerminal start screen

- [x] Typing `@` in the start-screen textarea opens a file-search dropdown populated from the workspace file tree (`useFileStore`); if the tree is empty when `@` is first typed, `fetchTree()` is called to load it
- [x] While the file tree is loading (`treeLoading === true`), the dropdown shows a loading indicator (e.g. "Loading filesâ€¦") rather than an empty list
- [x] With no query (just `@`), the dropdown lists the first 50 files from the tree; with a query it fuzzy-searches by filename (weight 0.7) and path (weight 0.3), threshold 0.3, up to 50 results; the first item is highlighted by default and the selection resets to index 0 whenever results change
- [x] The dropdown is keyboard-navigable: ArrowDown/ArrowUp moves the highlighted item, Enter inserts the selected file's full workspace-relative path, Escape closes the dropdown without inserting
- [x] Clicking a dropdown item inserts `@full/path` at the cursor position in the textarea, replacing the `@query` text, and moves the cursor to just after the inserted path
- [x] The dropdown closes when `onChange` fires and `findAtPosition` returns null (i.e. the `@` character was deleted or the cursor is now before it); explicit cursor movement without typing does **not** close the dropdown
- [x] The dropdown is positioned above the textarea when there is insufficient space below, otherwise below; positioning uses `getBoundingClientRect()` against `window.innerHeight` and must not render outside the modal's visible bounds
- [x] `Cmd/Ctrl+Enter` always submits the form immediately without inserting the highlighted file, whether or not the dropdown is open; pressing Enter while the dropdown is open inserts the highlighted file and does **not** submit
- [x] Pasting text that ends with an incomplete `@query` token opens the dropdown; pasting a complete `@path/to/file` (with a space or end-of-text after it) does not open the dropdown
- [x] File paths containing spaces are inserted verbatim; the `@query` regex stops at whitespace so they cannot be further refined by typing after insertion (known limitation, no fix required)
