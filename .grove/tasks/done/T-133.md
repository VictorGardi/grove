## Description

When a task is in "doing" column with an active agent terminal (tmux session active and opencode open), the injection prompt ("Please read X for your task context and instructions") appears repeatedly — essentially every time the agent becomes idle after doing work.

### Root Cause

T-119 added `readFreshFrontmatter` to fix stale Zustand store issues, but when the function returns `null` (due to malformed YAML, regex edge cases, or I/O errors), the code falls back to the stale store value (`task.terminalExecContextSent`), which is `false` — triggering re-injection on every reconnect/mount.

Additionally, `contextSentRef` is not reset on the reconnect path, and there's no validation that we're reconnecting to the correct session.

### Fix Required

1. When `readFreshFrontmatter` fails/returns null, skip re-injection (treat as "unknown" rather than using stale value)
2. Make the regex robust against edge cases: quoted values, trailing comments, whitespace variations
3. Add session validation: verify the session in frontmatter is alive before reconnecting; if not, fall back to the session from props
4. Reset `contextSentRef` on reconnect to ensure clean state
