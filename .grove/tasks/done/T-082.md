## Description

Allow users to type text in the chat input field when an agent is running (in either plan or execution mode). Currently, the input field is completely disabled (`disabled={isRunning}`) while the agent is active. The user should be able to compose text, but sending should remain blocked until the agent finishes.

**Files to modify:**

- Input field is NOT disabled when agent is running - user can type
- Send button remains disabled/cancelled when agent is running - cannot send
- Queued text can be sent once agent finishes
- Placeholder text can remain "Waiting for agent..." or show queued text count
