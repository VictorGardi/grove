## Description

Add terminal-style history navigation to the chat input. When pressing **Arrow Up** while the input is focused (and no dropdown is open), cycle backward through the messages the user has previously sent in this chat session. **Arrow Down** cycles forward. The current selection is shown in the input field, allowing the user to edit and re-send.

This is scoped to each task's chat session only — messages sent in other tasks are not included. A "chat session" is defined as the period since the task's chat was opened (terminal created for this task).

### Edge Cases

- If no messages have been sent in this session, Arrow Up/Down does nothing
- At the oldest message, Arrow Up does nothing (does not loop)
- At the newest message (or empty input), Arrow Down does nothing
- Only user-sent messages are included (not AI responses)

## Definition of Done

- [x] Pressing Arrow Up when input is focused and no dropdown is open displays the most recent message the user sent in the input field
- [x] Continuing to press Arrow Up cycles backward through all user messages in this chat (newest → oldest)
- [x] Pressing Arrow Down cycles forward (oldest → newest)
- [x] At the oldest message, Arrow Up does nothing (history position unchanged)
- [x] At empty input, Arrow Down does nothing
- [x] When input has been modified from the history selection, Arrow Up/Down continues to navigate the history relative to the currently displayed message
- [x] Pressing Enter sends the currently displayed text (whether from history or newly typed)
- [x] After sending a message, history position resets to "no selection" (empty input)
- [x] Arrow Up/Down does NOT interfere with existing autocomplete dropdowns — when any dropdown is open (e.g., @mention autocomplete, file path autocomplete), Arrow Up/Down should not trigger history navigation
- [x] Empty input field shows newest message on first Arrow Up press
- [x] If no messages have been sent in this session, Arrow Up/Down does nothing
