## Description

**Root Cause**: When using task switcher (Cmd+K) to open a task in another workspace, the `setActiveWorkspace` call updates `activeWorkspacePath` to the new workspace before `setSelectedTask` is called. Inside `setSelectedTask`, it reads `activeWorkspacePath` (now the new workspace) but uses the task's original `filePath` (from the old workspace), triggering the path traversal check in `main/tasks.ts:437`.

**Fix**: Pass the task's `workspacePath` to `setSelectedTask` so it uses the correct path regardless of the current active workspace.

**Files to modify**:

1. `src/renderer/src/stores/useDataStore.ts` - Add optional `workspacePathOverride` parameter to `setSelectedTask`
2. `src/renderer/src/stores/useTaskSwitcherStore.ts` - Pass `sortedTask.workspacePath` to `setSelectedTask`
