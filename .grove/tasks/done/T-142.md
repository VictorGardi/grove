## Description

Add status badges to the task switcher (Cmd+Shift+K) to make task status (backlog, doing, review, done) visible.

**Current behavior:**

- `isRunning = true` → shows `●` (accent color)
- `status === "review"` → shows `✓` (success color)
- Everything else → shows `○` (gray/idle)

**Proposed changes:**

1. **Add status badges**: Display status as a pill-shaped badge next to each task
   - backlog → gray badge
   - doing → blue (accent) badge
   - review → amber/orange badge
   - done → green (success) badge

2. **Enhance running indicator**: Keep `●` for running tasks, add "running" text label

3. **Layout**: Show both status badge AND running indicator when applicable

Example:

```
[T-142] Fix login bug           [doing] ● running
[T-143] Refactor API            [done] ✓
[T-144] Investigate bug         [backlog]
```

## Definition of Done

- [ ] Add status badge rendering for all 4 statuses (backlog, doing, review, done)
- [ ] Use theme-consistent colors (accent for doing, success for done, etc.)
- [ ] Preserve existing running indicator (●) and add "running" text
- [ ] Layout handles long task titles without breaking
- [ ] Works across all themes (default, evergreen, tokyo-night, etc.)
- [ ] Keyboard navigation (↑↓ Enter) still works correctly
