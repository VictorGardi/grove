## Description

The task switcher ordering has inconsistent MRU (Most Recently Used) behavior. When opening the switcher without a search, tasks viewed >1 hour ago get the same priority as never-viewed tasks due to a hard "1 hour" threshold in `groupSort` (lines 206-217 in `useTaskSwitcherStore.ts`). This breaks the ability to quickly switch back-and-forth between two tasks like Cmd+Tab on macOS.

### Root Cause Analysis

In `getSortedTasks` (lines 141-246 of `src/renderer/src/stores/useTaskSwitcherStore.ts`):

1. **groupSort** uses a hard cutoff:
   - Viewed within 1 hour → groupSort = 2
   - Currently running → groupSort = 1
   - Otherwise (viewed >1h ago or never) → groupSort = 0

2. **tier** in sortScore also has a 24-hour cutoff (lines 178-184)

This creates "cliffs" where:

- Task viewed 59 min ago: top priority
- Task viewed 61 min ago: same as never-viewed

### User Experience Impact

- Cmd+Tab style switching is broken when switching between tasks viewed >1 hour apart
- Inconsistent ordering feels random rather than predictable
- Users can't rely on spatial memory for frequently-switched tasks
