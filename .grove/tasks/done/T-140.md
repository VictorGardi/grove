## Description

### Problem

User sees duplicate tasks in the task switcher (Cmd+Shift+K). Duplicates:

- Appear in the SAME workspace (e.g., "datainfra-shared")
- Are EXACT copies (same ID, title, status)
- Only appear when the user SEARCHES (no duplicates on initial open)
- User sees ~4 duplicates of certain tasks

### Investigation findings

- `useAllTasksStore` maintains a `Map<workspacePath, TaskInfo[]>` of all tasks
- `getSortedTasks()` iterates workspaces and their tasks to build display list
- React renders with key = `task.task.id` (no workspace in key)
- Initial load works fine, duplicates appear when searching

### Root causes identified

1. **React key collision**: TaskSwitcherModal used `key={task.task.id}` without workspace in key, causing React reconciliation issues when duplicates existed
2. **Race condition**: Both App.tsx and TaskListInSidebar.tsx subscribed to `data.onChanged` - fixed by removing subscription from TaskListInSidebar
3. **Missing deduplication**: getSortedTasks didn't deduplicate by (task.id, workspacePath) - added safety net

### Fixes applied

1. Added deduplication in `getSortedTasks()` using `{workspacePath}:{task.id}` key
2. Fixed React key in TaskSwitcherModal: `key={task.task.id}` → `key={task.workspacePath}:{task.task.id}`
3. Removed duplicate `data.onChanged` subscription from TaskListInSidebar.tsx (line 37-45) - only App.tsx should listen for data changes
