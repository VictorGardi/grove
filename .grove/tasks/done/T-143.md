## Description

Add an intermediate "Starting agent session" status indicator that displays immediately when a task is moved to "doing" and the terminal starts initializing, before transitioning to "Agent is running" once the agent produces its first output.

**Current behavior:**

- Task moved to "doing" → terminal starts → shows "Waiting for input" immediately
- Only transitions to "Agent is running" when specific keywords detected ("esc+interrupt" for opencode, "Thinking" for copilot)
- No visual feedback during the startup window (typically 1-10 seconds)

**Expected behavior:**

- Task moved to "doing" → terminal starts → shows "Starting agent session" (with loading spinner)
- When first content chunk arrives (session_id or any output) → transitions to "Agent is running"
- If no content after 10 seconds timeout → reverts to "Waiting for input" (terminal still usable)

## Definition of Done

- [x] New status "Starting agent session" appears immediately when task moved to "doing" and terminal session begins
- [x] Status uses visual loading indicator (orange/spinner) distinct from green "running" and yellow "waiting"
- [x] Status transitions to "Agent is running" when first content chunk arrives from agent
- [x] If no agent output within 10 seconds, status reverts to "Waiting for input" (terminal remains usable)
- [x] Existing "Agent is running" detection (keyword-based) continues to work as before when agent is active
