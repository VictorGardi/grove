## Description

The current `ToolUseBlock` displays all tool inputs as raw JSON dicts, regardless of tool type. This means `edit` calls show `{"filePath": "...", "oldString": "...", "newString": "..."}` instead of a readable diff, and `bash` calls show `{"command": "..."}` instead of the command itself. The goal is to replace generic JSON dumps with per-tool semantic renderers — each tool type gets a purpose-built display that extracts meaning from the input keys and presents it professionally.

### 1. Per-tool semantic renderers (ToolUseBlock.tsx)

Each tool gets its own header layout and expanded view:

- **edit**: Header shows file icon + basename prominently + directory path in muted text + `+N -N` line count badge. Expanded view shows a visual diff: removed lines highlighted red with `-` prefix, added lines highlighted green with `+` prefix. Truncate diffs longer than 200 lines with "show more". Fallback to `data.title` if `filePath` is absent.
- **write**: Header shows file icon + basename + directory path. Expanded shows file content with syntax highlighting via Shiki, language inferred from file extension (`.tsx`, `.py`, etc.) with plain text fallback for unknown extensions.
- **read**: Header shows file icon + basename + directory path. No input JSON shown. Expanded shows output content (file contents) only.
- **bash**: Header shows the `command` string as the title (not `{"command": "..."}`). Expanded shows ANSI-colored output as before.
- **glob / grep**: Header shows the `pattern` value as title. Expanded shows file list output.
- **task**: Header shows subagent type as a label pill + `description` as subtitle. Expanded shows output.
- **webfetch / websearch**: Header shows URL as title. Expanded shows output.
- **Default fallback**: All other tools continue to use the current JSON input display.

Per-tool renderers must not conflict with `data.title` — use `data.title` only as fallback when the semantic field is absent.

### 2. Todo list custom checkboxes

Replace the native `<input type="checkbox" disabled>` (which renders as an OS-styled grayed-out control) with custom SVG checkboxes that match the app's design system. Completed items should have a checkmark SVG in accent color; pending items show an empty circle/square in border color.

### 3. Preserve existing features

Syntax highlighting (Shiki), copy button, line numbers, ANSI rendering, collapsible behavior, and scroll containers are already implemented — do not regress them.
