## Description

Tasks in the review column (and all columns) are incorrectly showing both "agent running" AND "waiting for input" indicators simultaneously. These are mutually exclusive states - only one should display at a time.

**Root Cause:**  
In `TaskCard.tsx`, the `isAgentRunning` flag (line 62) includes `execTmuxAlive`:

```javascript
const isAgentRunning = execIsRunning || execTmuxAlive || reviewerIsRunning;
```

However, "waiting for input" (`isExecuteWaitingWithTmuxAlive`) also checks for `execTmuxAlive` being true. When the agent has exited but tmux is still alive, both conditions become true, causing both badges to render.

**Fix:**  
Remove `execTmuxAlive` from `isAgentRunning` and `planTmuxAlive` from `isPlanningRunning`. The agent is only "running" when the actual agent process is active (`execIsRunning`/`planIsRunning`), not merely when the tmux session is alive.
