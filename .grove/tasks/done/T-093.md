## Description

When a user switches model mid-conversation using `/model`, the header shows the new model correctly but the model displayed next to agent messages in the chat still shows the previous model.

**Root Cause:**

- Agent messages in `PlanMessage` interface have no `model` field
- `ChatMessage` component receives `sessionModel` (from `session?.model`) instead of the model that was active when each message was created
- When `/model` is used, the store's `session.model` updates, but old messages lose their original model context

## Definition of Done

- [x] Add `model` field to `PlanMessage` interface in `src/shared/types.ts`
- [x] Capture model in `startAgentMessage` action in `usePlanStore.ts`
- [x] Update `ChatMessage` component to use `msg.model` instead of global `sessionModel`
- [x] Model shown next to each agent message reflects the model active when that message was created
- [x] Header and message model displays match after `/model` switch mid-conversation
