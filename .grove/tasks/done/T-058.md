## Description

When an agent finishes executing in the "doing" column and NOT all DoD boxes are checked, no indicator is currently shown on the task card. This task adds a "Waiting for input" indicator for this state.

Additionally, unify the backlog indicator label from "waiting for you" to "Waiting for input" (same logic and styling as currently is).

### Indicator Precedence (doing column)

1. Agent running → "agent running"
2. Reviewer running → "review pending"
3. Agent done + DoD complete → "ship it 🚢"
4. **Agent done + DoD incomplete → "Waiting for input" (NEW)**
5. Session failed → "session failed"

### Implementation

- File: `src/renderer/src/components/Board/TaskCard.tsx`
- Add condition: `task.status === "doing" && isExecuteWaiting && !isDodComplete && !isReviewerRunning`
- Reuse existing `waitingRow`, `waitingDot`, `waitingLabel` CSS classes
- Change backlog label from "waiting for you" to "Waiting for input" (line 164)
