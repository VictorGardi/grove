## Description

When changing task status to "done" via the **dropdown select** in TaskDetailPanel or TaskCentricView, tmux sessions are NOT killed (unlike drag-to-done on the board which correctly kills them).

### Root Cause

- **TaskDetailPanel.tsx** (line 500-502): The `else if (newStatus !== task.status)` branch only calls `moveTask(task.filePath, newStatus)` - no kill logic.
- **TaskCentricView.tsx** (lines 209-213): The `else` branch only calls `doMoveTask(selectedTask.filePath, newStatus)` - no kill logic.
- **Board.tsx** (lines 485-507): The `handleDragToDone` correctly kills both sessions and handles worktree teardown.

### Fix Required

In both `handleStatusChange` functions, add same logic as drag-to-done when `newStatus === "done"`:

1. **Kill exec session**: Call `taskterm.kill` with ptyId (`taskterm-exec-{taskId}`) and sessionName (`movedTask.terminalExecSession`), then update frontmatter to set `terminalExecSession: null`
2. **Kill plan session**: Call `taskterm.kill` with ptyId (`taskterm-plan-{taskId}`) and sessionName (`movedTask.terminalPlanSession`), then update frontmatter to set `terminalPlanSession: null`
3. **Worktree teardown**: If task has worktree, show confirmation dialog (same as Board.tsx), then call `git.teardownWorktreeForTask`. If it fails, rollback task move.
4. **Match drag behavior exactly**: Show confirmation dialog before removing worktree; rollback if teardown fails.

### Files to Modify

- `src/renderer/src/components/TaskDetail/TaskDetailPanel.tsx` - `handleStatusChange` function (lines ~415-505)
- `src/renderer/src/components/TaskCentric/TaskCentricView.tsx` - `handleStatusChange` function (lines ~117-214)
