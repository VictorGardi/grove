## Description

### Issue 1: Cmd+E not working

The keyboard shortcut for switching to the previous task (Cmd+E) does nothing when pressed. The current implementation at `useKeyboardShortcuts.ts:52-67` has flawed logic:

- It tries to find a task that isn't `lastTaskId`, but this doesn't correctly implement "toggle between current and previous"
- There's already a `toggleLastTask` function in `useTaskSwitcherStore.ts:100-125` that properly implements toggle behavior but isn't being used

### Issue 2: queueChunk import error (blocking)

`App.tsx:16` imports `queueChunk` from `usePlanStore.ts`, but this function doesn't exist in the store. This causes a Vite build error that breaks hot reloading and may prevent the app from loading properly.

### Issue 3: Duplicate key React warning

Console shows: `Encountered two children with the same key, /Users/victor/test/grove:T-172`. Components using `key={task.id}` can create duplicates when the same task ID exists in multiple workspaces.

### Issue 4: ShortcutsModal missing Cmd+E

The help modal at `ShortcutsModal.tsx` doesn't show Cmd+E as a available shortcut.
