## Description

Create and manage opencode sessions tied to Grove tasks. This layer maps `taskId → sessionId`, sends prompts with task context, and provides message history for UI replay. No streaming yet — that is T-202.

Depends on: T-200 (server must be running before sessions can be created).

## Implementation

### Dependency

Verify and add the correct package name for the opencode SDK (the task description suggests `@opencode-ai/sdk/v2` — confirm the exact npm package name and import path before starting). Add to `package.json` dependencies.

### `src/main/opencode/client.ts` — SDK client factory (NOT in src/shared/)

`src/shared/` is type-only code bundled for both processes. The SDK uses Node-only APIs and must only live in `src/main/`.

```ts
// Module-level cache: one client per server URL
let cachedClient: { url: string; client: OpenCodeClient } | null = null;

export function getClient(serverUrl: string): OpenCodeClient {
  if (!cachedClient || cachedClient.url !== serverUrl) {
    cachedClient = { url: serverUrl, client: createClient({ baseUrl: serverUrl }) };
  }
  return cachedClient.client;
}
```

### `src/main/ipc/opencodeSession.ts` — IPC handlers

In-memory session map (no file persistence — sessions are re-discovered on startup in T-204):

```ts
const sessions = new Map<string, { sessionId: string; worktreePath: string; serverUrl: string }>();
```

**`opencodeSession:create({ taskId, worktreePath })`**
1. Call `ensureServer()` to get `serverUrl`
2. Call `client.session.create({ directory: worktreePath })` → get `sessionId`
3. Store in `sessions` map
4. Write `execSessionId` to task frontmatter via `taskService.patchTask(taskId, { execSessionId: sessionId })`
5. Return `{ sessionId }`

**`opencodeSession:prompt({ taskId, promptText })`**
- Look up `sessionId` from map (error if not found)
- Call `client.session.prompt({ sessionID, parts: [{ type: 'text', text: promptText }] })`
- `promptText` is passed as a **pre-built string from the renderer** — the renderer constructs it using existing `buildFirstPlanMessage()` / `buildFirstExecutionMessage()` from `src/renderer/src/utils/planPrompts.ts`. The main process handler does not import renderer modules.

**`opencodeSession:stop({ taskId })`**
- Look up `sessionId` from map
- Call the appropriate SDK interrupt/abort method (verify exact API shape against SDK types)
- Return `{ ok: true }`

**`opencodeSession:get({ taskId })`**
- Returns `{ sessionId, status }` from the in-memory map
- Does NOT return message history — that is `opencodeSession:messages`

**`opencodeSession:messages({ taskId })`**
- Look up `sessionId` from map
- Call `client.session.messages({ sessionID })` → returns full message/parts array
- Return the messages array for UI hydration on remount
