## Description

Add a vertical drag handle to the right edge of the sidebar so users can resize it horizontally. The implementation should follow the existing pattern in `TerminalPanel.tsx` (mousedown/mousemove/mouseup on window, clamped min/max, localStorage persistence). Width is managed via React state and applied as an inline style — replacing the current `width: var(--sidebar-width)` rule in `Sidebar.module.css` (the CSS variable becomes a default/fallback only).

## Definition of Done

- [x] A vertical drag handle (4px wide) is rendered on the right edge of the sidebar
- [x] Cursor changes to `col-resize` when hovering the drag handle
- [x] `document.body.style.cursor` is set to `col-resize` for the duration of a drag and reset to `''` on mouseup (prevents cursor flicker when mouse moves faster than the handle)
- [x] Sidebar width can be dragged between a minimum of 160px and a maximum of 480px
- [x] Width is persisted to localStorage (key: `grove:sidebar-width`) and restored on remount; defaults to 240px when no stored value exists
- [x] Layout remains stable (no text overflow, no broken flex) across the full 160px–480px range
- [x] The drag handle has a higher z-index than the terminal panel overlay so it remains accessible
- [x] Sidebar toggle (Cmd+B) continues to work after resizing; on re-show, the persisted width is restored correctly
