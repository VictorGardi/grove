## Description

The agent sometimes stops mid-task without completing. The user sees the chat history but the "Running" indicator disappears, and no error or failure state is shown. The conversation appears cut off. The user can resume by typing a new message, but has no visibility into what caused the previous run to stop or whether the tmux session is still alive.

This happens during both planning and execution modes, appearing random and several times per session.

## Definition of Done

- [ ] When agent stops unexpectedly, the UI shows a clear failure state (not just "not running") — either exit code, error message, or warning indicator
- [ ] User can see what happened — exit code is visible and/or error message is shown in the chat
- [ ] Session state is properly tracked: when a session dies without the `grove_exit` sentinel, `lastExitCode` is set to indicate failure (not left as null which implies "success/pending")
- [ ] Debug tab shows accurate session state including whether tmux session is alive

## Context for agent

### Relevant code paths:

1. **Agent execution flow:**
   - `src/main/planManager.ts` — spawns agent (or uses tmux via TmuxPlanRunner)
   - `src/main/tmuxSupervisor.ts` — runs agent in tmux, tails log file, detects `grove_exit` sentinel
   - Log file at `~/.grove/runs/<session>.log` — contains all agent output
   - Tailer uses `fs.watch` + 50ms poll interval to read new lines

2. **Chunk processing:**
   - Main process sends chunks via `plan:chunks` IPC (micro-batched per event loop turn)
   - `src/renderer/src/App.tsx` line ~150 — receives chunks and routes to PlanStore
   - `src/renderer/src/stores/usePlanStore.ts` — `applyChunk` updates state, `done` chunk sets `isRunning: false`

3. **Session state:**
   - `PlanSession.isRunning` — true when agent message starts, false on `done` chunk
   - `PlanSession.lastExitCode` — set from `done` chunk content
   - `PlanSession.sessionStatus` — "running", "paused", "reconnecting", "idle"

### Investigation notes:

- The `grove_exit` sentinel (written by shell script in `tmuxSupervisor.ts:317`) signals normal completion
- If tmux dies without writing this sentinel, no `done` chunk is emitted
- The running indicator could be cleared via: (a) `done` chunk, (b) `replay_done` chunk, (c) `setRunning(false)` in various effects
- Check if `lastExitCode` is null when this happens (indicates crash vs graceful exit)
- Consider adding logging to detect when/why `isRunning` transitions to false

### Edge cases to test:

- tmux session terminates without writing sentinel
- Log file deleted during tailing
- Network/disk issues during write
