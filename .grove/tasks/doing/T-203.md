## Description

Replace the xterm.js terminal with a structured React event feed for opencode tasks. This is the only renderer-side task in the series. It wires up the execution flow in `executionActions.ts`, renders typed SDK events as rich components, and handles remount/replay via message history.

Depends on: T-200, T-201, T-202.

## Implementation

### Execution flow — `src/renderer/src/actions/executionActions.ts`

When `agent === "opencode"`:
1. Skip tmux session creation entirely
2. `await api.opencodeServer.ensure()` — get `serverUrl`
3. `await api.opencodeSession.create({ taskId, worktreePath })` — get `sessionId`
4. Build prompt in renderer: `buildFirstExecutionMessage(task, rawContent)` (plan mode: `buildFirstPlanMessage`)
5. `await api.opencodeSession.prompt({ taskId, promptText })`
6. Update `terminalExecContextSent` in task frontmatter for traceability (existing field)
7. Subscription is started by `<TaskEventStream>` on mount — do NOT call `api.opencodeEvents.subscribe` from the actions layer

Stop button: calls `api.opencodeSession.stop({ taskId })`.

When `agent !== "opencode"`: existing tmux/PTY path unchanged.

### `src/renderer/src/components/TaskDetail/TaskEventStream.tsx`

Main container. Props: `{ taskId: string; mode: 'plan' | 'execute' }`.

**Mount lifecycle:**
1. `useEffect` on mount:
   a. Call `api.opencodeSession.messages({ taskId })` → hydrate local state (handles remount mid-stream)
   b. Call `api.opencodeEvents.onEvent(taskId, handleEvent)` → store unsubscribe ref
2. `useEffect` cleanup: call stored unsubscribe function

**Local state:**
```ts
type Message = { id: string; parts: Part[]; role: 'assistant' | 'user'; done: boolean }
type Part = TextPart | ToolCallPart | ToolResultPart
const [messages, setMessages] = useState<Message[]>([]);
const [sessionStatus, setSessionStatus] = useState<'idle' | 'busy' | 'retry'>('idle');
const [permissionRequest, setPermissionRequest] = useState<PermissionEvent | null>(null);
```

**`handleEvent(events: OpenCodeEvent[])`** — processes the batched event array from T-202:
- `message.part.updated`: find or create message by ID; update the part (replace by part index/ID)
- `session.status`: set `sessionStatus` from `event.properties.type`
- `session.error`: push an error entry into messages
- `permission.asked`: set `permissionRequest` state
- Unknown types: log to console, do not throw

**Render:**
```tsx
<div className="opencode-event-stream">
  {messages.map(msg => <EventMessage key={msg.id} message={msg} />)}
  <SessionStatusBar status={sessionStatus} />
  {permissionRequest && <PermissionDialog event={permissionRequest} taskId={taskId} />}
</div>
```

### `src/renderer/src/components/TaskDetail/EventMessage.tsx`

Renders one message. Iterates `message.parts`:
- `part.type === 'text'` → `<StreamingText text={part.text} isStreaming={!message.done} />`
- `part.type === 'tool-call'` → `<ToolCallCard part={part} />`
- `part.type === 'tool-result'` → `<ToolResultCard part={part} />`
- Unknown part types → `<code>{JSON.stringify(part)}</code>`

### `src/renderer/src/components/TaskDetail/StreamingText.tsx`

Critical performance decision: **do not call `react-markdown` on every streaming event**.

```tsx
function StreamingText({ text, isStreaming }: { text: string; isStreaming: boolean }) {
  if (isStreaming) {
    // Cheap path: no parsing, no VDOM diffing beyond text update
    return <pre className="whitespace-pre-wrap font-sans">{text}</pre>;
  }
  // Finalized: full markdown render once
  return (
    <ReactMarkdown remarkPlugins={[remarkGfm]} components={{ code: SyntaxHighlightedCode }}>
      {text}
    </ReactMarkdown>
  );
}
```

Wrap the `isStreaming → false` transition in `startTransition` so the markdown parse doesn't block input.

Shown as an inline block (not a modal) when `permissionRequest` is set:


- "Deny" → `api.opencodeSession.respondPermission({ taskId, ..., decision: 'deny' })`
