## Description

Three polish concerns that require the full stack (T-200‚ÄìT-203) to be in place first:

1. **Startup reconciliation**: detect orphaned opencode sessions (Grove closed mid-run) and offer reconnect/discard
2. **Agent selector**: surface opencode in the task launch UI with auto-detection
3. **Worktree teardown cleanup**: stop the event subscription and clear session state when a task completes

Depends on: T-200, T-201, T-202, T-203.

## Implementation

### 1. Startup reconciliation

**Timing constraint**: Main-process startup runs before the renderer window loads. Do NOT attempt dialog/notification before the window signals readiness. Defer reconciliation until `mainWindow.webContents.on('did-finish-load')` fires (or the renderer sends a `renderer:ready` IPC message).

**Flow (in `src/main/opencode/reconcileOrphanSessions.ts`):**

```
On did-finish-load:
  1. Find all tasks in doing/ with execSessionAgent === 'opencode' and execSessionId !== null
  2. Try ensureServer() ‚Äî if server can't start, skip reconciliation and log
  3. For each such task:
     a. Call client.session.get({ sessionID: task.execSessionId })
     b. If 404 or error ‚Üí clear execSessionId in task frontmatter (taskService.patchTask)
     c. If found and status === 'busy' or 'idle':
        - Store in session manager Map (T-201)
        - Send mainWindow.webContents.send('opencode:orphan', { taskId, sessionId, status })
```

**Renderer side (add to `src/renderer/src/stores/usePlanStore.ts` or a new `useOpencodeStore.ts`):**

Listen for `opencode:orphan` events. For each:
- Show a non-blocking toast/banner on the task card (not a blocking modal): "opencode session resumed" or "opencode session active"
- If status === 'busy': immediately call `api.opencodeEvents.subscribe(taskId)` to resume the stream
- If status === 'idle': mark session as reconnected, let user continue with another prompt

**Preload**: add `api.opencode.onOrphan(cb)` following the push-subscription pattern.

### 2. Agent selector ‚Äî availability check

**Where the check lives**: main process only. The renderer cannot run `which opencode`.

Extend the existing `plan:listModels` IPC handler (`src/main/ipc/plan.ts`):
- The handler already calls `spawnOpencodeModels` which fails gracefully if opencode is not installed
- Cache the result: `opencodeAvailable: boolean` in module-level state after first check
- Expose via existing channel or a new `plan:getAgentCapabilities` ‚Üí `{ opencode: boolean, copilot: boolean, claudeCode: boolean }`

**UI (in the agent selector dropdown ‚Äî `TaskTerminal.tsx` or wherever the agent dropdown lives):**
- Fetch `api.plan.getAgentCapabilities()` on first render (cache in store)
- Show "opencode" option only if `capabilities.opencode === true`
- Default selection logic: if opencode available ‚Üí default to "opencode"; else ‚Üí "claude-code" (existing default)
- The auto-default only applies on first task start, not when the user has explicitly chosen an agent

### 3. Worktree teardown cleanup

**In `src/main/ipc/git.ts` ‚Äî `git:teardownWorktreeForTask` handler:**

Before removing the worktree, if `task.execSessionAgent === 'opencode'`:
1. Call `opencodeEvents.unsubscribe({ taskId })` (imported from `src/main/ipc/opencodeEvents.ts`)
2. Call `opencodeSession.remove({ taskId })` ‚Äî removes from in-memory session map
3. Patch task frontmatter: `{ execSessionId: null, execSessionAgent: null }`

**In `completeTask` flow (wherever the task is moved to done/):**

Same cleanup: unsubscribe events, clear session map entry, null out session frontmatter fields.

Note: Do NOT kill the opencode server on teardown. The server is shared across tasks; it stays alive until Grove quits.

## Definition of Done

- [ ] Reconciliation runs after `did-finish-load`, not before (no race with window load)
- [ ] Tasks in `doing/` with `execSessionAgent === 'opencode'` are checked against the live server on startup
- [ ] If a session no longer exists server-side (404), `execSessionId` is cleared from task frontmatter
- [ ] If a session is still active server-side, it is re-registered in the session map and the renderer is notified
- [ ] If the server fails to start during reconciliation, the reconciliation is skipped gracefully (no crash)
- [ ] A `busy` orphaned session immediately resumes event streaming in the renderer
- [ ] Reconciliation notification is non-blocking (toast/banner, not a modal blocking app startup)
- [ ] `plan:getAgentCapabilities` (or equivalent) returns `{ opencode: boolean }` based on a main-process `which opencode` check
- [ ] Agent selector shows "opencode" only when `capabilities.opencode === true`
- [ ] "opencode" is auto-selected as the default agent when available; falls back to "claude-code"
- [ ] Auto-default does not override a user's explicit agent choice
- [ ] `git:teardownWorktreeForTask` unsubscribes event stream and clears session map entry for opencode tasks
- [ ] `completeTask` clears `execSessionId` and `execSessionAgent` in task frontmatter for opencode tasks
- [ ] The shared opencode server is NOT killed on worktree teardown (only on Grove quit)
