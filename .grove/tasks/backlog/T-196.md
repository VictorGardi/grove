## Description

Add OS-level sandboxing for all three agents (claude, opencode, copilot) on macOS using Apple Seatbelt (`sandbox-exec`). This is Part 3 of the original T-192 task.

## Requirements

### Scope
- **macOS only.** Uses Apple Seatbelt (`sandbox-exec`). Linux support (bubblewrap) is out of scope for this task.
- On Linux, agents launch without any sandbox wrapping regardless of the `sandboxEnabled` setting.
- Sandboxing is **enabled by default** but can be disabled via workspace setting.

### Strategy per agent (macOS)

| Agent | Mechanism | Additional |
|-------|-----------|------------|
| `claude` | Claude Code's built-in sandbox via `.claude/settings.json` with `sandbox: { enabled: true }` in worktree root | Verify `sandbox.enabled` is the correct current Claude Code `settings.json` API |
| `opencode` | `sandbox-exec -f <profile>` wrapping the `opencode` binary invocation inside `bash -c` | Also set `external_directory: "deny"` in opencode config for defense-in-depth |
| `copilot` | `sandbox-exec -f <profile>` wrapping the `copilot` binary invocation inside `bash -c` | Investigate per-task session isolation |

### Critical implementation detail
The sandbox must wrap the agent binary, not the surrounding bash process:
```bash
bash -c "sandbox-exec -f /tmp/grove-sandbox-<sessionId>.sb opencode run ..."
```
NOT `sandbox-exec ... bash -c "opencode ..."` (which sandboxes bash and may or may not inherit to children).

### New module: `src/main/sandbox/profileGenerator.ts`
- `getSandboxPrefix(agent, worktreePath, sessionId)` â†’ returns the `sandbox-exec` command prefix string and cleanup callback; returns `null` on non-macOS (no-op)
- SBPL profile for opencode/copilot allows: worktree (rw), agent binary + XDG config/data/cache dirs, `/opt/homebrew/`, `/usr/`, `/System/`, `/Library/`, git config (ro), `$TMPDIR`; denies all other home directory reads
- Temp `.sb` profile files named `grove-sandbox-<sessionId>.sb`
- Grove scans and removes stale profiles at startup to handle abnormal exits

### opencodeConfig.ts redesign
Currently it overwrites the config file. Must instead read-patch-write (merge Grove's settings into existing user config) and on cleanup **restore the original content** rather than delete the file. This prevents destroying a user's existing opencode configuration.

### .claude/settings.json placement
Decide whether to write to worktree root (risk: committed to branch) or workspace's `.claude/` directory (outside worktree). If writing to worktree, add `.claude/settings.json` to workspace `.gitignore`.

### macOS sandbox-exec deprecation
`sandbox-exec` is deprecated on macOS 15 (Sequoia) but remains functional. Document this risk. If Apple removes it, migration path is bubblewrap-for-mac via Homebrew.

### Workspace-level toggle
Add `sandboxEnabled` boolean (default: `true`) to `WorkspaceEntry`. Expose in workspace settings UI as on/off toggle with description. When disabled, no sandbox wrapping or `.claude/settings.json` config applied. On Linux, this toggle has no effect.

### Worktree required when sandbox enabled on macOS
Task with `useWorktree: false` cannot start an agent when sandbox is on. Show clear error explaining sandboxing requires a worktree. When sandbox is disabled, worktree restriction doesn't apply.
