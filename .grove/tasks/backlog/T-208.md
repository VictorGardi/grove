# Make tool call UI match openchamber

## Problem
Current Grove tool call UI is bulky and expanded by default. Openchamber has a much more compact design where tool calls are collapsed by default and take minimal space.

## What openchamber does differently
Based on openchamber screenshots and description:
1. **Tool calls collapsed by default** - only expand when user clicks
2. **Compact header** - shows tool icon, name, summary, and status in one line
3. **No auto-expand** - even while running, the tool card stays collapsed
4. **Minimal space** - args/output only visible when explicitly expanded
5. **Timer shows inline** - live duration appears next to status, not taking extra space

## Current Grove behavior (wrong)
- `ToolCallCard.tsx:141` - `useState(isInFlight)` auto-expands while running
- Takes lots of vertical space with args + output visible
- No compact collapsed state design

## Plan

### 1. Change default expansion behavior
**File:** `src/renderer/src/components/TaskDetail/ToolCallCard.tsx:141`
- Change `useState(isInFlight)` to `useState(false)` 
- Tool calls start collapsed always
- Only expand on user click

### 2. Make collapsed header more informative (compact)
**File:** `src/renderer/src/components/TaskDetail/ToolCallCard.tsx`
- Show tool icon + name + summary + status in header
- Add elapsed timer inline when running
- Make entire header compact (smaller padding, smaller font)

### 3. Improve expanded content
**File:** `src/renderer/src/components/TaskDetail/ToolCallCard.tsx`
- Args section: collapsible with `<details>` tag
- Output: use rich renderers (already exist: GrepOutput, GlobOutput, etc.)
- Read file output: add syntax highlighting based on file extension

### 4. CSS improvements for compact look
**File:** `src/renderer/src/components/TaskDetail/TaskEventStream.module.css`
- Reduce padding/margins in `.toolCallCard`, `.toolCallHeader`, `.toolCallBody`
- Make collapsed state visually lighter
- Ensure expanded state is clearly differentiated

## Files to modify
