## Description

The opencode chat UI is functional but visually rough compared to reference implementations like openchamber. This task brings it up to a polished, professional level through four focused sub-tasks. No structural rewrites — all changes are confined to the TaskDetail chat components and their CSS module.

## Implementation

### Sub-task A — Fix CSS design tokens (foundation, do first)

**Why first:** Nine CSS variables used throughout `TaskEventStream.module.css` are undefined in the theme system, causing silent fallbacks (wrong/invisible colours on status badges, buttons, role labels). Nothing else looks right until this is fixed.

**Change: `TaskEventStream.module.css` only** — add a token-alias block at the top of the `.wrapper` rule. No theme files need to be touched.

```css
.wrapper {
  /* Alias undefined chat vars to the design system tokens */
  --text-muted: var(--text-lo);
  --bg-subtle: var(--bg-elevated);
  --border-subtle: var(--border-dim);
  --accent-blue: var(--status-blue);
  --accent-green: var(--status-green);
  --accent-yellow: var(--status-amber);
  --status-yellow: var(--status-amber);
  --accent-hover: color-mix(in srgb, var(--accent) 80%, white 20%);
  --accent-green-hover: color-mix(in srgb, var(--status-green) 70%, black 30%);
  /* rest of existing rules unchanged */
}
```

**Also fix: `src/renderer/src/components/Files/MarkdownViewer.tsx`** — it hardcodes `"grove-dark"` as the shiki theme instead of reading `useThemeStore((s) => s.colors.shikiTheme)`. Fix this as a byproduct since we're touching shiki infrastructure.

**Also add:** Scoped markdown prose styles in `TaskEventStream.module.css` under `.textPart` so rendered markdown (paragraphs, lists, blockquotes, inline code) looks consistent with prose elsewhere in the app. Reference `src/renderer/src/components/Files/MarkdownViewer.module.css` for the rules — scope them as `.textPart :global(p)`, `.textPart :global(ul)`, etc.

### Sub-task B — Syntax highlighting in chat code blocks

**Change: `src/renderer/src/components/TaskDetail/StreamingText.tsx`**

Replace the current no-op `SyntaxHighlightedCode` component with a `ChatCodeBlock` component that:

1. Receives `lang` from `className?.replace("language-", "")` and `code` from stringified `children`
2. Imports `getHighlighter` from `../../Files/shikiHighlighter` (lazy singleton, already warm when Files view was used)
3. Uses `useThemeStore((s) => s.colors.shikiTheme)` as a dep so code re-highlights on theme change
4. Calls `getHighlighter()` in a `useEffect` with a cancellation flag (same pattern as `MarkdownViewer.tsx` lines 22–46)
5. Renders token spans with `style={{ color: token.color }}` inside a styled `<pre>`
6. Adds a copy button: `<button onClick={() => navigator.clipboard.writeText(code)}>Copy</button>` positioned top-right of the block
7. Falls back to unstyled `<pre>` while loading (no flicker)
8. **Only activated when `!isStreaming`** — the existing `showMarkdown` gating in `StreamingText.tsx` already handles this

Wire via the `components` prop of `ReactMarkdown`: `components={{ pre: ChatCodeBlock }}` (wrap `pre`, not `code`, to get the full block including the copy button wrapper).

Add `.chatCodeBlock`, `.chatCodeHeader`, `.chatCopyButton` classes to `TaskEventStream.module.css`.

### Sub-task C — Tool call visual improvements

**Change: `src/renderer/src/components/TaskDetail/ToolCallCard.tsx`**

**Step 1 — Tool header summary.** Add a pure function `getToolSummary(toolName: string, state: ToolState): string`:
- Check `(state as { title?: string }).title` first — opencode populates this on `running` and `completed` states; use it directly if non-empty
- Fall back to a `switch (toolName)` with safe optional chaining for common tools:
  - `bash` / `execute_bash` → first line of `state.input?.command` truncated to 60 chars
  - `read_file` / `cat` / `view` → `state.input?.path ?? state.input?.file_path`
  - `write_file` / `create_file` → same path fields
  - `edit_file` / `str_replace_based_edit` / `str_replace` → same path fields
  - `glob` / `find` → `state.input?.pattern`
  - `grep` / `search` → `state.input?.pattern ?? state.input?.query`
  - Default → `""` (show tool name only)
- Render as a secondary `<span>` in the header, truncated, muted colour

**Step 2 — Tool type icons.** Plain `Record<string, string>` constant at top of file. `bash`/`execute_bash` → `$`, `read_file`/`cat`/`view` → `≡`, `write_file`/`create_file` → `+`, `edit_file`/`str_replace*` → `~`, `glob`/`find` → `*`, `grep`/`search` → `/`, `todo_read`/`todo_write` → `✓`, default → `⚙`. Render as `.toolCallIcon` span before the tool name — replacing the current `▶` chevron (keep the chevron as the expand indicator in a separate span).

**Step 3 — Terminal-styled bash output.** When `part.tool === "bash" || part.tool === "execute_bash"` and status is `completed`, apply an additional CSS class `.toolOutputTerminal` to the output section: `background: var(--bg-surface); color: var(--status-green); font-family: var(--font-mono)`. CSS-only change, no structural HTML changes.

**Animated BusyDots**

Create a minimal `BusyDots` component (inline in `TaskEventStream.tsx` or as `BusyDots.tsx`):
```tsx
function BusyDots(): React.JSX.Element {
  return (
    <span className={styles.busyDots}>
      <span className={styles.dot} style={{ animationDelay: "0ms" }}>•</span>
```

```css

  0%, 80%, 100% { opacity: 0.2; }

.dot { animation: busyDot 1.4s infinite; display: inline-block; margin: 0 1px; }
```
Replace the `"Thinking..."` span with `<BusyDots />`. Keep the `waitingBadge` span text unchanged.

**Scroll-to-bottom fix** (also fixes a pre-existing bug where SSE updates yank the user back)

In `TaskEventStream.tsx`:
- Add `const [isUserScrolled, setIsUserScrolled] = useState(false)` 
- Add `onScroll` to `.eventContainer` div: if `scrollHeight - scrollTop - clientHeight > 60` set `isUserScrolled(true)`, else `setIsUserScrolled(false)`
- Fix the auto-scroll `useEffect` to only call `scrollIntoView` / `scrollTo` when `!isUserScrolled`
- Render a FAB `<button>` when `isUserScrolled`: `position: sticky; bottom: 12px; align-self: flex-end` inside the container, clicking it calls `containerRef.current.scrollTo({ top: containerRef.current.scrollHeight, behavior: "smooth" })` and sets `isUserScrolled(false)`
- Add `.scrollFab` to `TaskEventStream.module.css`
