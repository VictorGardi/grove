## Description

Integrate opencode's SDK (`@opencode-ai/sdk/v2`) as a first-class execution backend for Grove tasks, replacing the tmux+TUI path for opencode agents. Each worktree gets its own `opencode serve` process. Sessions persist across Grove restarts. Grove streams opencode's typed events (~40 types) into a rich React UI instead of raw xterm.js terminal output.

## Implementation

### 1. opencode server lifecycle (main process)

`src/main/opencodeServerManager.ts` — manages one `opencode serve` per worktree:
- `ensureServer(worktreePath)` — spawns `opencode serve` if not running; auto-detects already-running instances via HTTP health check on port 4096
- `killServer(worktreePath)` — stops the process
- `listServers()` — returns running instances
- Instances stored in a `Map<worktreePath, { process, port }>`
- Cleanup on worktree removal and Grove shutdown
- Handles port conflict (fallback port) and restart-on-crash

`src/main/ipc/opencodeServer.ts` — IPC handlers: `opencodeServer:ensure`, `opencodeServer:kill`, `opencodeServer:list`

### 2. SDK client factory

- Add `@opencode-ai/sdk/v2` as a dependency
- `src/shared/opencode/client.ts` — factory creating `createOpencodeClient({ baseUrl })` per worktree, cached by worktree path
- Typed SDK client reused by session and event IPC handlers

### 3. Session management (main process)

`src/main/ipc/opencodeSession.ts` — IPC handlers keyed by `taskId`:
- `opencodeSession:create({ worktreePath, taskId })` — calls `client.session.create({ directory: worktreePath })`, stores session ID in `Map<taskId, sessionInfo>`
- `opencodeSession:prompt({ taskId, userText, taskRawContent, sessionMode, agent })` — injects context by sending the task context as the first message part, then userText as second part. Uses existing `buildFirstPlanMessage` / `buildFirstExecutionMessage` from `planPrompts.ts` as the prompt body
- `opencodeSession:stop({ taskId })` — calls `client.session.stop()` to interrupt the running session (Ctrl-C equivalent)
- `opencodeSession:get({ taskId })` — returns session state (running/idle, id)
- **Startup adoption**: on app launch, for each running server, call `client.session.list()` and reconcile active sessions with tasks in `doing/` state. Prompt user to reconnect or discard orphaned sessions.

### 4. Event streaming bridge (main → renderer)

`src/main/ipc/opencodeEvents.ts` — SSE → IPC relay:
- `opencodeEvents:subscribe({ worktreePath, taskId })` — calls `client.event.subscribe({ directory: worktreePath })`, iterates the SSE stream, sends each event to renderer via IPC channel `opencode:events:${taskId}`
- `opencodeEvents:unsubscribe({ taskId })` — cancels the async iterator and removes the subscription
- Each `taskId` gets a dedicated IPC channel to avoid multiplexing conflicts
- Reconnection: if the SSE stream errors, retry with exponential backoff up to 3 times before surfacing an error to the renderer
- `ipcRenderer.on` in renderer uses the `taskId` channel, dispatches events to the appropriate `usePlanStore` / `useTaskStore` slice

### 5. Rich event renderer (replaces xterm.js for opencode tasks)

`src/renderer/src/components/TaskDetail/TaskEventStream.tsx`:
- Receives typed event objects from the IPC bridge
- Renders each event type as a rich component:
  - `message.part.updated` → `StreamingMarkdown.tsx` (live character-by-character markdown rendering)
  - `message.done` → marks task as complete, updates exit code
  - `tool.use` → `ToolCallCard.tsx` (collapsible — tool name + args)
  - `tool.result` → `ToolResultCard.tsx` (collapsible output, syntax highlighted)
  - `session.idle` → hides busy indicator, enables input
  - `session.busy` → shows spinner in terminal area
  - `error` → `ErrorBanner.tsx`
  - Unknown types → raw `code` block with event type label
- `StreamingMarkdown.tsx` — uses `react-markdown` with streaming character appends, updates on each `message.part.updated` event
- `usePlanStore` / `useTaskStore` accumulation: the renderer accumulates `message` events into the existing message array. If the renderer remounts mid-stream, it requests a replay of the current session state from the main process via `opencodeSession:get`.

### 6. Execution flow (modify existing)

`src/renderer/src/actions/executionActions.ts`:
- When `agent === "opencode"`, skip tmux/tmux-session creation entirely
- Sequence: `opencodeServer:ensure(worktreePath)` → `opencodeSession:create({ worktreePath, taskId })` → `opencodeSession:prompt({ taskId, userText, ... })` → `opencodeEvents:subscribe({ worktreePath, taskId })`
- Mount `<TaskEventStream taskId={taskId} />` in the task panel instead of `<TaskTerminal />`
- `terminalExecContextSent` task field still updated for traceability
- Stop button calls `opencodeSession:stop({ taskId })`

### 7. Agent selector

- Add `"opencode"` option in the task launch modal alongside `"claude-code"` and `"copilot"`
- Auto-select `"opencode"` as default if `@opencode-ai/sdk` resolves and `opencode` binary is found; fall back to `"claude-code"` otherwise
