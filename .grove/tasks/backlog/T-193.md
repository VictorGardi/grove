## Description

When Claude Code is running in Grove's task terminal (plan/exec mode), scrolling to navigate the terminal history doesn't work as expected. Instead, when Claude Code shows a question with selectable options, scrolling scrolls through those options rather than moving through the terminal's scrollback buffer.

**Root cause**: Claude Code's TUI captures scroll input when it has an active prompt with choices, using it to navigate between options. This prevents the user from scrolling up/down to see previous terminal output.

**Comparison with other agents**:
- Copilot: Shows options but does NOT capture scroll for navigation — scroll works for terminal history
- OpenCode: Shows options but does NOT capture scroll for navigation — scroll works for terminal history
- Claude Code: Captures scroll input to navigate options — prevents terminal scrolling

**Technical limitation**: This is a terminal TUI behavior issue. xterm.js correctly forwards mouse wheel events as escape sequences to the PTY when the terminal application requests them. Claude Code's TUI uses mouse wheel for option navigation — this cannot be overridden from Grove without upstream changes.

**Possible approaches (to evaluate during implementation)**:
1. Check if Claude Code has a flag to disable mouse-based option navigation
2. Add visual indicator when scroll is captured by an interactive prompt
3. Add documentation/tooltip explaining behavior difference
4. Add scrollback preview panel feature for all agents

## Definition of Done

- [ ] Evaluate if Claude Code can be configured to not capture mouse scroll (e.g., CLI flag)
- [ ] User can scroll up/down to view previous terminal output while Claude Code is NOT showing an interactive prompt with options
- [ ] IF scroll capture persists during prompts: Add visual indicator/badge showing "scroll captured by prompt"
- [ ] Document behavior difference in UI or help (so users understand why scroll behaves differently)
