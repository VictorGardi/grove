## Description

Renew Grove's visual identity with **"Grove's Forest"** — a design language derived from the app's own name and nature metaphor. Replace reliance on the borrowed Catppuccin/Tokyo Night aesthetic with a Grove-native identity built from a custom forest-dark palette. Combine this with a **per-session color identity system** from the "Signal" proposal: 8 nature-named accent colors (Fern, Moss, Bark, Stone, Water, Slate, Amber, Clay) assigned to agent sessions at creation and carried through the sidebar, task cards, terminal tabs, and task detail panel.

### Design Direction

**Forest dark palette (new default theme):**
- `--bg-base: #0c1010` · `--bg-surface: #131a15` · `--bg-elevated: #1c2520`
- `--bg-hover: #243028` · `--bg-active: #2a3830`
- `--border: #2e3d35` · `--border-dim: #1e2d25`
- `--text-primary: #e2ede2` · `--text-secondary: #98ab98` · `--text-lo: #5a6b5a`
- `--accent: #3d9970` (forest green) · `--status-amber: #e8a838`
- `--status-green: #52b788` · `--status-red: #e05c5c` · `--status-blue: #4d96b8`

**Session color palette (8 nature-named colors, round-robin assignment):**
| Name  | Hex       |
|-------|-----------|
| Fern  | `#52b788` |
| Moss  | `#3d9970` |
| Bark  | `#a0785a` |
| Stone | `#7a8c99` |
| Water | `#4d96b8` |
| Slate | `#6b7fa3` |
| Amber | `#e8a838` |
| Clay  | `#c47356` |

Session colors are propagated to components via a `--session-color` CSS custom property set as an inline style on the task list item / card root element. Child elements reference it via `var(--session-color, var(--accent))` with `--accent` as fallback. Session colors are fixed hex values independent of the active theme — this is intentional and documented in code.

**Running session animation (single behavior):**
Task cards with an actively running agent show the session-color left border with a breathing pulse animation: `@keyframes sessionPulse` — opacity oscillates 1.0 → 0.5 → 1.0 over 1.8s ease-in-out infinite. No separate top-border strip.

**Typography:**
- Base font size: 13px → 14px (xterm.js terminal font size is explicitly excluded — managed separately)
- Sidebar section labels: uppercase 10px → lowercase 11px

**New design tokens:**
- `--radius-xl: 12px` — applied to task cards and dialogs
- `--transition-spring: 160ms cubic-bezier(0.34, 1.56, 0.64, 1)` — applied to task card hover transforms

**Migration:**
- Forest becomes the default theme for new installs / first launches only (no stored `theme` preference)
- Existing stored user theme preferences are preserved without migration

**Out of scope for this task (separate tasks):**
- Meadow (light) and Dusk (warm dark) themes — palettes undefined, defer
- Timeline tab in TaskDetail — requires new agent activity data source / IPC work
- Bento home/dashboard view — requires separate scoping

## Context for agent

The existing design system lives in `src/renderer/src/styles/`. Theme files are in `src/renderer/src/styles/themes/`. The theme is applied via `data-theme` attribute on `<html>` and managed by `src/renderer/src/stores/useThemeStore.ts`. Base tokens (layout, radii, transitions, fonts) are in `src/renderer/src/styles/variables.css`.

Task metadata is managed through the task store. Session color should be added as a new optional field on the task type in `src/shared/types.ts` and populated at task creation time.

All component CSS uses CSS modules (`.module.css` files co-located with components). The session color CSS custom property approach avoids needing to pass hex values through props — set it once at the container level and all child CSS module files can reference `var(--session-color)`.
