## Description

Enable task cards in the review column to show all indicators that backlog/doing columns can show, using the same logic and conditions. Currently, review column tasks don't show any status indicators (agent running, waiting, error, etc.) — they should show the same indicators as doing/backlog when the conditions are met.

Currently in TaskCard.tsx:

- "agent running" shown for: `doing` (execIsRunning) or `backlog` (isPlanningRunning)
- "review pending" shown for: `doing` (reviewerIsRunning)
- "ship it 🚢" shown for: `doing` (DoD complete + reviewer passed)
- "Waiting for input" shown for: `doing` (isExecuteWaiting) or `backlog` (isPlanWaiting)
- "session failed" shown for: `doing` (isExecuteErrored) or `backlog` (isPlanErrored)
- Branch badge shown for: `doing`

The review column should show all these same indicators using identical logic, just with `task.status === "review"` added to each condition.

## Definition of Done

- [x] TaskCard.tsx: Add `task.status === "review"` to the condition for "agent running" indicator (should show when reviewerIsRunning OR execIsRunning OR execTmuxAlive)
- [x] TaskCard.tsx: Add `task.status === "review"` to the condition for "Waiting for input" indicator (should show when isExecuteWaiting or isPlanWaiting)
- [x] TaskCard.tsx: Add `task.status === "review"` to the condition for "session failed" indicator (should show when isExecuteErrored or isPlanErrored)
- [x] TaskCard.tsx: Add `task.status === "review"` to the condition for branch badge (show branch when task.branch exists)
- [x] Run typecheck to verify the change compiles correctly

Note: "ship it" and "review pending" are not applicable to review column since they relate to moving from doing → review → done flow.
