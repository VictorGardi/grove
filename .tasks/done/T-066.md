## Description

Add two new sections to README.md that explain the Grove workflow and its value to developers.

### "How It Works" section

A step-by-step narrative placed **before** the existing "Features" section:

1. Define tasks as Markdown in `.tasks/`, decisions in `.decisions/`, milestones in `.milestones/` — no separate tool needed
2. Drag a task to **Doing** (or press `D`) → Grove automatically creates a git worktree and isolated branch
3. Run any CLI AI agent (Claude Code, Copilot CLI, Codex, Aider, OpenCode) in the embedded terminal — no API key setup in the app. Use in-app chat for ticket refinement
4. Review what the agent changed via the built-in diff view
5. Move to **Done** → Grove prompts to clean up the worktree

### "Why Grove" section

Benefits to place after "Principles", reworded to avoid duplication:

- **No context switching** — task board, terminal, file browser, and diff review in one window
- **Isolated work** — one worktree per task means clean parallel development without manual branch switching
- **No lock-in** — everything is Markdown files in your repo; delete the app, your tasks remain

(Other benefits — no auth, repo as source of truth, no API key management — are already covered by the existing Principles section.)
