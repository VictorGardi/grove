## Description

### Feature 2: Last-Task Toggle Shortcut

**Trigger**: `Cmd+Shift+Tab` (to avoid OS-level Cmd+Tab conflict)

**Behavior**:

- Toggles between current task and previous task
- Maintains history stack: `[mostRecent, secondMostRecent, ...]`
- Ignores duplicate consecutive entries

**On toggle**:

- Same as Feature 1 select behavior

**Edge cases**:

- History empty or only one task → no-op
- Previous task's workspace deleted → pop and try next
- Previous task no longer exists → pop and try next

---

### Feature 3: Active Task Tabs / Strip

**UI Placement**: Horizontal strip at top of MainArea, above content

**Content**:

- Currently active task (always shown)
- 4 most recently viewed tasks
- Tasks with running agents (surfaced to front)

**Each tab displays**:

- Task ID (e.g. T-004)
- Status indicator: ● running / ✓ review / ○ idle

**Behavior**:

- Click tab → instant switch (same as palette)
- Order: active task first, then recency, with running tasks pinned to front
- Close button on hover (removes from strip, does NOT delete task)

**Hover tooltip**:

- Full task title
- Workspace name

---

### State Management

**New Store: useTaskSwitcherStore**

```typescript
{
  isOpen: boolean;
  searchQuery: string;
  recentTaskIds: string[];  // max 10 items
  lastTaskId: string | null;
  taskLastViewedAt: Record<string, number>;  // taskId -> timestamp
}
```

**Persisted between reloads**: None (in-session only per user request)

**Integration**:

- When task is viewed (selected), update recentTaskIds and taskLastViewedAt
- Track running agents via existing execSessionId field on tasks
- Use terminalStore.tabs to detect active sessions

---

### Integration with Agent System

- Detect running agent via task.execSessionId (non-null = running)
- On switch to task with running session:
  - Open terminal panel
  - Reattach to tmux session (via existing supervisor)
  - Show live output
- Task in review → optionally show Diff tab
- Task idle → show last used view (Agent or Task)
