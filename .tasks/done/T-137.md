## Description

Two issues with Cmd+Shift+K task switcher:

### Issue 1: Sorting uses priority tiers with tmux session check

Current: Running tasks (execSessionId only) get +10000, then lastViewedAt, then created.
Problem: Tasks with active tmux sessions should be at the top.

Fix in useTaskSwitcherStore.ts getSortedTasks:

- Check BOTH terminalPlanSession AND terminalExecSession (not just execSessionId)
- Use priority tiers sorting:
  - Tier 1: Active tmux session (terminalPlanSession OR terminalExecSession exists)
  - Tier 2: Recently viewed (lastViewedAt within last 24h)
  - Tier 3: Older viewed (lastViewedAt > 24h ago)
  - Tier 4: Never viewed (sort by created, most recent first)

Edge case: Check if tmux session is alive before prioritizing (use taskterm.isAlive).

### Issue 2: Task titles showing as "New Task"

Root cause: parseTaskFile already has fallback to filename for "new task", but T-137 has title: "New task" in frontmatter.

Fix: Update T-137.md frontmatter to remove explicit title or use descriptive title from filename.

## Definition of Done

- [ ] Tasks with terminalPlanSession OR terminalExecSession appear at top in Cmd+Shift+K
- [ ] Recently viewed tasks (>24h ago) sort above never-viewed tasks
- [ ] Never-viewed tasks sort by created (newest first)
- [ ] Dead tmux sessions are handled gracefully (not placed at top)
- [ ] T-137 no longer shows "New Task" as title (use filename fallback)
