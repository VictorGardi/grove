## Description

Agent chat panel should allow switching between "Plan" and "Execute" conversation modes, instead of automatically selecting based on task status. This allows users to view and continue either conversation at any time.

**Key behavior:**

- Toggle/tab inside agent chat to switch between Plan and Execute modes
- Default to "Execute" if execution session exists (via `task.execSessionId` or messages in `execute:{taskId}` session), otherwise default to "Plan"
- Switching tabs shows different conversation history (stored messages for that session)
- Both tabs allow sending new messages that continue the respective conversation
- Toggle state is local UI state (not persisted)

**Edge cases:**

- When switching tabs mid-streaming (agent running), the response should be allowed to complete (or cancelled appropriately)
- Empty sessions should show appropriate placeholder text
- Scroll position should be handled appropriately when switching (auto-scroll to new messages, preserve if user scrolled up)
- Agent/model selections may need to persist per-mode or be managed appropriately
