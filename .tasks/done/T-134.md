## Description

### Problem

The task list in sidebar doesn't instantly update when:

- Moving a task to "done" via drag and drop
- Creating a new task

User must reload to see changes.

### Root Cause

The codebase has file watchers (`src/main/watchers.ts`) that send `workspace:dataChanged` IPC event when task files change. The preload exposes `window.api.data.onChanged(callback)` to subscribe. However, `TaskListInSidebar.tsx` doesn't subscribe to this event — it only fetches tasks on mount/workspace change.

### Proposed Fix

1. Add subscription to `workspace:dataChanged` event in `TaskListInSidebar.tsx`
2. When event fires, re-fetch tasks for all workspaces to update the displayed list

## Definition of Done

- [x] Sidebar task list updates within 2 seconds when a task is moved to done (drag and drop)
- [x] Sidebar task list updates within 2 seconds when a new task is created
- [x] No manual reload required to see task list changes
