## Description

The "New task" button lives in `BoardToolbar`, which is only rendered when `tasks.length > 0`. On a fresh workspace with no tasks, the user sees an empty state ("No tasks yet. Create a Markdown file in .tasks/backlog/ to get started") with no way to create a task via the UI.

**Fix:** Add a "Create first task" button to the empty state in `Board.tsx` that calls the existing `createTask("New task")` action. The backend `createTask` already calls `initTaskDirs()` before writing the file, so the `.tasks` directory structure will be created automatically.

**Files to change:**

- `src/renderer/src/components/Board/Board.tsx` — add button to the empty state (lines 219-230)
