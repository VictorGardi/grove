## Description

Refactor Grove to cleanly separate the core runtime from the UI layer. The goal is to make Grove architecture-driven instead of UI-driven. Task lifecycle, worktree management, container execution, agent launching, task state, and session attachment must live in a shared runtime layer that can be used by both the Electron app and a future CLI.

### Core requirement

- The Electron app must become a thin UI over the runtime.
- The CLI must also be able to use the same runtime later.
- No task execution logic should live only inside UI components.

### Design goals

- UI should only render state, trigger actions, and display logs/status.
- Runtime should own all operational logic:
  - creating tasks
  - resolving task paths
  - creating optional worktrees
  - starting/stopping execution environments
  - launching/attaching to agents
  - tracking active sessions
  - reading/writing task markdown files
  - managing task status transitions

### Architecture boundary

- **UI layer**: React/Electron views, modals, task page, sidebar, switcher, file viewer.
- **Runtime layer**: pure TypeScript modules or services with no UI/Electron dependencies.

### Implementation approach

1. **Create runtime package at `src/runtime/`**
   - Pure TypeScript with no Electron imports
   - Uses `child_process`, `fs`, `path`, `simple-git`, standard libraries only
   - Has its own tsconfig (tests run with vitest without Electron)

2. **Runtime API surface**

   **Task operations:**
   - `createTask(workspacePath, title) -> TaskInfo`
   - `listTasks(workspacePath) -> TaskInfo[]`
   - `updateTask(workspacePath, filePath, changes, body?) -> TaskInfo`
   - `moveTask(workspacePath, filePath, toStatus) -> TaskInfo`
   - `archiveTask(workspacePath, filePath) -> void`
   - `readTaskBody(workspacePath, filePath) -> string`
   - `readTaskRaw(workspacePath, filePath) -> string`
   - `writeTaskRaw(workspacePath, filePath, rawContent) -> TaskInfo`
   - `resolveTaskPath(workspacePath, taskId) -> string | null`
   - `initTaskDirs(workspacePath) -> void`

   **Worktree operations:**
   - `createWorktreeForTask(workspacePath, taskId, taskTitle) -> SetupWorktreeResult`
   - `teardownWorktreeForTask(workspacePath, worktreePath) -> void`
   - `listWorktrees(repoPath) -> WorktreeInfo[]`
   - `detectWorktreeBaseBranch(repoPath, worktreePath) -> string | null`
   - `listBranches(repoPath) -> BranchInfo[]`
   - `getDiff(repoPath, branch?) -> DiffSummary`
   - `getFileDiff(repoPath, filePath, branch?) -> string`
   - `readFileAtBranch(repoPath, filePath, branch) -> string`

   **Execution operations:**
   - `startExecution(taskId, mode, message, agent, model, options) -> void`
   - `stopExecution(taskId, mode) -> void`
   - `attachToSession(taskId, mode) -> Promise<boolean>`
   - `getExecutionState(taskId) -> ExecutionState | null`
   - `listActiveSessions() -> ActiveSession[]`
   - `listModels(workspacePath) -> string[]`

   **Session management:**
   - `getTaskSession(taskId, mode) -> SessionInfo | null`
   - `saveTaskSession(taskId, mode, sessionId, agent, model) -> Promise<void>`
   - `clearTaskSession(taskId, mode) -> Promise<void>`

3. **Files to move to runtime**
   - `src/main/tasks.ts` → `src/runtime/taskService.ts`
   - `src/main/git.ts` → `src/runtime/gitService.ts`
   - `src/main/tmuxSupervisor.ts` → `src/runtime/tmuxService.ts`
   - `src/main/agentRunner.ts` → `src/runtime/agentService.ts`
   - `src/main/contextGenerator.ts` → `src/runtime/contextGenerator.ts`
   - `src/main/fileWriter.ts` → `src/runtime/fileWriter.ts`
   - `src/main/agentOutputParser.ts` → `src/runtime/agentOutputParser.ts`

4. **Integration**
   - `src/main/ipc/` handlers become thin wrappers that call runtime APIs
   - Renderer actions continue to call IPC (no changes needed there)
   - For CLI: runtime is imported as a library

5. **Existing tests**
   - Tests in `src/main/__tests__/` already run without Electron
   - Add more tests in `src/runtime/__tests__/` as needed
