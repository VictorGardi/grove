## Description

Users cannot see content in the Edit tab or Changes tab in the TaskDetailPanel. When they click on these tabs, the content doesn't appear.

### Root Cause

A useEffect at lines 271-274 in `src/renderer/src/components/TaskDetail/TaskDetailPanel.tsx` forces the tab back to "plan" every time a task is selected:

```tsx
useEffect(() => {
  if (!task) return;
  setActiveTab("plan");
}, [task?.id, setActiveTab]);
```

This effect overrides any manual tab selection the user makes.

### Fix

Remove this useEffect. The `activeTab` is already initialized from `workspaceBoardStates[workspacePath]?.taskDetailTab` (defaulting to "edit"), which provides correct default behavior without overriding user selections.
