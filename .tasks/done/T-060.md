## Description

Replace the current "bubble" chat metaphor with a full-width, document-style layout inspired by Claude Code, Copilot Chat, and modern terminal-based AI assistants. The current implementation uses 85% max-width bubbles with left/right alignment that change shape based on content length, creating visual inconsistency.

### Changes to `PlanChat.module.css`:

1. **Remove bubble constraints**: Change `.message` max-width from 85% to 100%, remove `align-self` rules that position messages left/right, all messages should be full-width and left-aligned
2. **Distinguish roles through subtle styling**: User messages get a subtle accent tint (`color-mix(in srgb, var(--accent) 5%, transparent)`), agent messages get transparent background, remove borders from bubbles, increase border-radius to `var(--radius-lg)`
3. **Add visual separation**: Each message gets `border-bottom: 1px solid var(--border-dim)` and `padding-bottom: 12px` for consistent separation
4. **Improve readability**: Add `max-width: 75ch` on text content for readability at wide panel sizes, add `max-height: 400px` with `overflow-y: auto` on code blocks
5. **Role label accessibility**: Keep role labels visually but make them subtler, add `aria-label` attributes for screen readers

### Changes to `PlanChat.tsx`:

1. Add `role="article"` and `aria-label` attributes to message containers
2. Ensure all existing functionality works: streaming cursor, thinking blocks, file autocomplete (`@`), model switching (`/model`), cancel/send buttons, reconnect UI, exit warnings
3. Verify placeholder messages, error states, and session status banners render correctly in new layout
