## Description

### Root Cause

When a new task is created with an agent session (terminalPlanSession/terminalExecSession), the status indicator doesn't appear on the board because `useWorkspaceStatus.ts` (lines 54-73) fetches tasks once on initial load but never updates when new tasks are created. It only re-runs when `workspacePath` changes, so the polling loop checks liveness against stale task data.

The main `useDataStore` correctly gets the new task via `patchTask()`, but `useWorkspaceStatus` has its own separate task state that never sees it.

### Proposed Fix

Subscribe `useWorkspaceStatus.ts` to `useDataStore` instead of fetching tasks separately. This ensures the liveness polling always operates on current task data.
