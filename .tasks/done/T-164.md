## Description

Optimize the Grove Electron app by addressing 6 performance-related issues identified during code review.

## Issues to Fix

### Issue 1: Excessive Debug Logging

- **Files**: `useDataStore.ts`, `useTaskSwitcherStore.ts`, `TaskDetailPanel.tsx`, `taskActions.ts`
- **Problem**: 17+ `console.log/warn/error` statements fire on every interaction (task selection, workspace switch), creating noise and performance overhead
- **Fix**: Remove all `console.*` statements or guard behind `process.env.NODE_ENV === "development"`

### Issue 2: Duplicate Logic in useWorkspaceStatus

- **File**: `src/renderer/src/hooks/useWorkspaceStatus.ts` lines 202-258
- **Problem**: Two separate loops iterate over `relevantTasks` - once for aggregate status, once for count
- **Fix**: Combine into single pass tracking both status AND count simultaneously

### Issue 3: Missing React.memo on TaskCard

- **File**: `src/renderer/src/components/Board/TaskCard.tsx`
- **Problem**: TaskCard has 5 store subscriptions, no memoization - re-renders on any store change
- **Fix**: Wrap with `React.memo`, add `useCallback` for click handlers (lines 93-97) to prevent new function refs

### Issue 4: Unmemoized Filtered Tasks in Board

- **File**: `src/renderer/src/components/Board/Board.tsx` line 97
- **Problem**: `const filtered = searchResults ? searchResults.map(...) : tasks` NOT memoized - runs every render
- **Fix**: Memoize `filtered` with `[tasks, searchResults]` dependency using `useMemo`

### Issue 5: Redundant Render Tick

- **File**: `src/renderer/src/hooks/useWorkspaceStatus.ts` lines 162-165, 262
- **Problem**: `setRenderTick` is redundant - liveness store subscription (line 70) already triggers re-renders
- **Fix**: Remove `setRenderTick` and render-tracker, rely on `liveness` subscription alone

### Issue 6: Unstable Handler Reference

- **File**: `src/renderer/src/hooks/useKeyboardShortcuts.ts` line 22
- **Problem**: `handleKeyDown` is defined inside useEffect, recreated each run
- **Fix**: Wrap `handleKeyDown` in `useCallback` with `[]` deps since it only uses `getState()`
