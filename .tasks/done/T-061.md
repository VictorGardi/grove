## Description

When editing a task's markdown file in the Edit tab, the autosave (debounced at 500ms) triggers a task patch via `useDataStore.getState().patchTask(result.data)` in `saveRaw`. This updates the task object in the store, which causes the `useEffect` at line 107-110 to re-run because `task` is in the dependency array. The effect then calls `setActiveTab("plan")`, switching the user back to the Agent tab.

The `useEffect` that defaults to the Agent tab currently has `[task?.id, task]` as dependencies. Including the full `task` object means it fires on every task property change, not just when a new task is selected.

**Fix:** Change the dependency array from `[task?.id, task]` to `[task?.id]` so the default tab is only set when the task ID changes (i.e., when opening a different task), not on every task update within the same task.

Note: This means reopening the same task will preserve the last-used tab rather than resetting to Agent. This is acceptable behavior.
