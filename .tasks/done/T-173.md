## Description

**Root Cause**: When creating a task in another workspace via TaskSwitcherModal, `taskActions.ts` calls `setSelectedTask(result.data.id)` without passing the workspace path. The `setSelectedTask` function then uses the current active workspace path (wrong workspace) with the new task's filePath (in the other workspace), triggering a path traversal error when trying to read the task body.

**Fix**: Modify `src/renderer/src/actions/taskActions.ts` to:

1. Switch to the workspace where the task was created using `setActiveWorkspace(wp)`
2. Pass both `filePathOverride` and `workspacePathOverride` to `setSelectedTask`
