## Description

Build an htop-style **"Agent Sessions"** monitor view in Grove — a live, auto-refreshing dashboard showing all active Grove tmux sessions across all workspaces. The primary goal is making it immediately obvious when a tmux session lingers after its task is complete (a **HUNG** session), with a one-click kill to clean it up. Secondary: at-a-glance visibility into which agent/model is running, how long the session has been alive, and whether it is actively working or idle.

### Navigation

Add a new **"Agents"** nav item to the sidebar, placed between the terminal toggle and settings, using an activity-style icon from `lucide-react`. When ≥1 HUNG session exists, display a red count badge on this nav item (reusing the existing badge pattern). Clicking the item sets `activeView = "agents"`, rendered in `MainArea.tsx` by a new `<AgentMonitor />` component.

### Architecture

#### 1. New IPC handler — `tmux:listGroveSessions`

Add to `src/main/ipc/taskTerminal.ts` (or a new `src/main/ipc/tmuxMonitor.ts`). Wire it in `src/main/ipc/index.ts`, expose it in `src/preload/index.ts`, and add the type signature to `src/preload/index.d.ts`.

The handler:

1. Checks tmux availability via the existing `TmuxSupervisor.isTmuxAvailable()`. If tmux is absent, returns `[]`.
2. Runs `tmux list-panes -a -F "#{session_name}|#{pane_current_command}|#{pane_pid}|#{pane_dead}|#{session_created}|#{pane_activity}"` and deduplicates rows by `session_name` (take first row per session — Grove sessions have one pane each).
3. Filters to rows whose `session_name` starts with `grove-`.
4. Parses the session name using two regex patterns to extract `workspaceHash`, `taskId`, and `mode`:
   - Agent sessions: `/^grove-(plan|exec)-([a-f0-9]{6})-(.+)$/` → groups: mode, hash, taskId
   - Terminal sessions: `/^grove-term-([a-f0-9]{6})-(.+)-(plan|exec)$/` → groups: hash, taskId, mode
   - (Both must correctly handle task IDs containing hyphens, e.g. `T-42`.)
5. Iterates all known workspaces from `ConfigManager`, hashes each workspace path (SHA-256 hex, first 6 chars) to build a `hash → { workspacePath, workspaceName }` lookup map.
6. For each matched session, calls `scanTasks(workspacePath)` for the matching workspace and looks up the task by `taskId` to obtain: `taskStatus`, `agent`, `model`. If no task is found, these fields are `null` and the session is classified HUNG.
7. Computes `idleSeconds = now - paneActivityTs` and `durationSeconds = now - sessionCreatedTs`.
8. Returns `TmuxSessionInfo[]`.

#### 2. New `tmux:killSession` IPC handler

Add alongside the above (same wiring steps). Takes `{ sessionName: string }` and runs `tmux kill-session -t <sessionName>`. For `grove-term-*` sessions where a PTY is attached, the caller should also invoke `pty:kill` with the corresponding `ptyId` (`taskterm-<taskId>`) to avoid orphaned PTYs.

#### 3. `AgentMonitor` React component

New component at `src/renderer/src/components/AgentMonitor/AgentMonitor.tsx`.

- Polls `tmux:listGroveSessions` every 5 seconds via `setInterval`.
- Renders an htop-style table with columns: **STATUS · TASK · WORKSPACE · MODE · AGENT · MODEL · IDLE · DURATION · ACTIONS**.
- Status classification (color-coded dot + label), evaluated in priority order:
  - 🔴 **HUNG**: `taskStatus === 'done'` AND session is alive, OR `taskStatus === null` (orphaned — no matching task found). These are the sessions that should not exist.
  - 🟡 **LINGERING**: `taskStatus === 'review'` AND session alive (task is in review, session may still be wrapping up — softer warning).
  - 🟢 **RUNNING**: `taskStatus === 'doing'`, pane command is a known agent binary (`opencode`, `copilot`, `aider`, `claude`), idle < 30 s.
  - 🟡 **WAITING**: `taskStatus === 'doing'`, pane command is agent binary, idle ≥ 30 s (likely awaiting user input or a long tool call — cosmetic hint only).
  - ⚪ **IDLE**: `taskStatus === 'doing'`, pane command is a shell (`bash`, `zsh`, `sh`) — agent exited cleanly, session persists.
  - 🔴 **ZOMBIE**: `paneDead === true` (dead pane left in an alive session).
- Kill button on each row calls `tmux:killSession`. For `grove-term-*` sessions, additionally calls `pty:kill` with `ptyId = "taskterm-<taskId>"`. Optimistic UI: hide the row immediately; restore on next poll if kill failed.
- Clicking a task ID: if the session belongs to the currently active workspace, sets `activeView = "board"` and selects the task. If it belongs to a different workspace, calls `setActiveWorkspace(workspacePath)` first, then navigates to board.
- Empty-state handling: "No active sessions" when tmux returns no Grove sessions; "tmux is not available" when tmux is not installed; "No workspaces configured" when `ConfigManager` has no workspaces.
- A note at the top of the view acknowledges that this polling loop runs independently of the existing `useWorkspaceStatus` liveness polling — intentional duplication for MVP; consolidation deferred.

### What `TmuxSessionInfo` looks like

```ts
interface TmuxSessionInfo {
  sessionName: string;
  sessionType: "plan" | "exec" | "term-plan" | "term-exec";
  workspaceHash: string;
  workspacePath: string | null;
  workspaceName: string | null;
  taskId: string;
  taskStatus: "backlog" | "doing" | "review" | "done" | null;
  agent: string | null;
  model: string | null;
  paneCommand: string;
  panePid: number;
  paneDead: boolean;
  sessionCreatedTs: number; // Unix seconds (from #{session_created})
  paneActivityTs: number; // Unix seconds (from #{pane_activity})
  idleSeconds: number; // computed
  durationSeconds: number; // computed
}
```
