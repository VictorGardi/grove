## Description

When the user presses Cmd+Shift+K to open the task switcher, sometimes it doesn't work and shows an empty task list. Cmd+R (reload) fixes the issue.

**Root cause:** The task data in `useAllTasksStore.allTasks` is fetched asynchronously when workspaces load (see `App.tsx:52-58`). When `getSortedTasks` is called in the TaskSwitcherModal, if the task data hasn't been fetched yet for a workspace, the tasks for that workspace don't appear in the list. Since there's no loading indicator in TaskSwitcherModal (it shows "No tasks found" even when loading), users think the shortcut isn't working.

**Verification:** Check if `allTasks.get(workspacePath)` returns undefined/empty array when Cmd+Shift+K is pressed on fresh launch.
