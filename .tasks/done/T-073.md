## Description

When creating tasks in a fresh workspace, the task ID does not start at `T-001`. Instead, it continues from the highest ID generated in any previously opened workspace during the same app session.

**Root cause:** The `lastGeneratedId` variable in `src/main/tasks.ts:32` is a module-level singleton shared across all workspaces. When workspace A creates tasks up to `T-072`, then workspace B is opened, the counter is still at 72, so workspace B's first task becomes `T-073`.

**Fix:** Change `lastGeneratedId` from a single number to a `Map<string, number>` keyed by resolved workspace path, so each workspace maintains its own counter. Use `path.resolve(workspacePath)` as the key for path normalization.
