## Description

### Root Causes

1. **Race condition in idle detection** - `isIdle()` checks for 3 seconds of no PTY output (`pty.ts:223-226`), but this doesn't mean the agent is ready to read input. The agent may be waiting for model loading or initialization.

2. **Fixed 300ms delay before Enter** - `injectContext.ts:70` uses a fixed 300ms delay between writing the instruction and sending Enter. If the agent is still outputting startup messages at 300ms, the Enter key is lost or ignored.

3. **No injection verification** - `terminalExecContextSent` is set to `true` (`injectContext.ts:73`) without confirming the agent actually received/processed the context file.

### Proposed Fixes

1. **Add agent-ready handshake** - Inject a unique marker string, wait for it to appear in PTY output before sending actual context. This ensures the agent is in a state to process input.

2. **Add retry logic** - Retry injection with exponential backoff if no acknowledgment is received within the timeout.

3. **Add agent-specific detection** - Different ready signals for OpenCode vs Copilot vs Claude Code (each has different startup output).

4. **Improve logging** - Add timestamps for all injection stages to debug timing issues.

### Key Files

- `src/renderer/src/utils/injectContext.ts` - Main injection logic
- `src/main/pty.ts` - PTY management and `isIdle()`
- `src/main/ipc/taskTerminal.ts` - Terminal session creation
