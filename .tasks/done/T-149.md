## Description

**Root Cause:** In `switchToTask` (`src/renderer/src/stores/useTaskSwitcherStore.ts:231`), `setActiveWorkspace()` is called without `await`. Since it's async, `setSelectedTask()` runs immediately before the workspace actually switches, so the task lookup fails (searches wrong workspace's task list).

**Three locations have this bug:**

1. **Task switcher** (`useTaskSwitcherStore.ts:231`): `workspaceStore.setActiveWorkspace(sortedTask.workspacePath)` - not awaited
2. **Cmd+digit shortcuts** (`useKeyboardShortcuts.ts:179`): `setActiveWorkspace(ws.path)` - not awaited
3. **Sidebar task click** (`TaskListInSidebar.tsx:46-48`): Uses `setTimeout(..., 100)` workaround — race-prone antipattern

**Fix:**

1. In `useTaskSwitcherStore.ts`, await the `setActiveWorkspace` call before calling `setSelectedTask`. Also handle the case where `setActiveWorkspace` fails (API error) — don't call `setSelectedTask` if the workspace switch failed.

2. In `useKeyboardShortcuts.ts` (Cmd+digit handler around line 179), await `setActiveWorkspace` before any subsequent actions that depend on the new workspace.

3. In `TaskListInSidebar.tsx`, use async/await instead of `setTimeout`:
   ```typescript
   if (task.workspacePath !== activeWorkspacePath) {
     await setActiveWorkspace(task.workspacePath);
     setSelectedTask(task.id);
   } else {
     setSelectedTask(task.id);
   }
   ```
