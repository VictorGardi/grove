## Description

When using the Copilot agent in Grove, the agent visibly runs in the background (tmux terminal shows activity) but no responses appear in the chat panel. The root cause is a mismatch between what `parseCopilotLine()` in `src/main/agentOutputParser.ts` expects and what the Copilot CLI actually emits with `--output-format json --stream on`.

`parseCopilotLine` currently only handles four JSON message shapes: `session_id`, `message`(assistant), `delta`, and `assistant.message_delta`. If the CLI emits a different or additional shape, lines are silently dropped — no error, no UI update. The agent runs normally but the chat stays empty.

## Context for agent

The parser lives in `src/main/agentOutputParser.ts` — specifically `parseCopilotLine()`. The tmux supervisor tails `~/.grove/runs/<session>.log`; each line is a JSON object from the CLI. `parseCopilotLine` must return `PlanChunk[]` for each recognised line; unrecognised lines currently return `[]` (silent drop).

The Copilot CLI is invoked as:
```
copilot -p "<prompt>" --output-format json --stream on --no-color [--deny-tool=shell] [--allow-tool=write(<path>)] [--model=<model>]
```

Log file format: first line is `{"type":"grove_user_message","content":"..."}` (written by Grove), then raw CLI output lines, then `{"type":"grove_exit","code":0}` sentinel (written by the Grove shell script).

Key files:
- `src/main/agentOutputParser.ts` — parser to fix
- `src/main/tmuxSupervisor.ts` — log tailer that calls `parseCopilotLine`
- `src/main/planManager.ts` — spawn runner that also calls `parseCopilotLine`
- `src/main/__tests__/` — existing test suite (Vitest)
