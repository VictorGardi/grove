## Description

Add a "Files" tab to the Task Page for editing files within a task's worktree.

### UI Placement

Add new "Files" tab to Task Page, alongside existing "Agent" and "Changes" tabs.

### Layout

```
[ File Tree (left) ] | [ Editor (right) ]
```

### Components

1. **File Tree Panel** (`src/renderer/src/components/Task/Files/`)
   - Reuse existing file tree component (or adapt from FilesView)
   - Show files in task's worktree
   - Indicators for: unsaved (dirty), git-modified files

2. **Editor Panel** (`src/renderer/src/components/Task/Files/FileEditor.tsx`)
   - Use Monaco Editor (`@monaco-editor/react`)
   - Features: syntax highlighting, undo/redo, keyboard editing, scroll + selection
   - Track per-file state: content, isDirty, lastSavedTimestamp

3. **Backend** (`src/main/ipc/filesystem.ts`)
   - Add `fs:writeFile` IPC handler (validate path within worktree, write atomically)
   - Reuse existing `fs:readFile` for content loading

### Editing Model

```
File State:
- content: string (editor buffer)
- isDirty: boolean (unsaved changes)
- lastSavedTimestamp: Date
```

### Save Behavior

- Cmd/Ctrl+S triggers write to disk
- Updates lastSavedTimestamp, clears isDirty
- Returns success/error with message

### External Change Handling (CRITICAL)

Use file watching (existing `watchOpenFile` or chokidar):

- **File NOT dirty**: auto-reload from disk
- **File IS dirty**: show prompt with options:
  - "Reload" → replace buffer with disk content
  - "Keep My Changes" → keep buffer, ignore disk update

### Worktree Integration

- All reads/writes scoped to task's worktree
- No writes to main repo
- Enables safe, isolated editing per task

### State Indicators

In file tree:

- unsaved changes → highlight/marker
- git-modified → (optional)

### Scope (MVP)

**Include**:

- open file from tree
- edit text in Monaco
- save file (Cmd+S)
- file tree navigation
- external change detection + prompt

**Exclude**:

- extensions/plugins
- advanced refactoring
- multi-cursor
- full IDE features


### useTaskFileStore State Shape

```typescript
interface TaskFileStore {
  // For current task's worktree
  worktreePath: string;

  // Tree
  tree: FileTreeNode[];
  treeLoading: boolean;

  // Open files (single file for MVP)
  openFilePath: string | null;
  content: string; // current editor buffer
  savedContent: string; // last saved to disk
  isDirty: boolean;
  lastSavedTimestamp: Date | null;

  // Binary/large detection
  isBinary: boolean;
  isTooLarge: boolean;

  // Actions
  fetchTree(): void;
  openFile(path: string): Promise<void>;
  saveFile(content: string): Promise<void>;
  markDirty(content: string): void;
  reloadFromDisk(): Promise<void>;
}
```

### IPC Flow

```
Renderer                         Main Process
  |                                   |
  |── fs:readFile(wp, relPath) ──────>|
  |<── { ok, data: {content, lang} } ─|
  |                                   |
  |── fs:writeFile(wp, relPath, cnt) >|
  |<── { ok } or { ok: false, err } ─|
  |                                   |
  | (listen via fs:onFileChanged) <───|
```

### External Change Handling

- Main process already watches files via `watchOpenFile()` in `src/main/watchers.ts`
- Sends `fs:fileChanged` event to renderer
- In FilesTab: subscribe to `window.api.fs.onFileChanged` and check if changed file matches open file
- If isDirty: show modal with [Reload] [Keep My Changes]
- If !isDirty: auto-reload via store action
