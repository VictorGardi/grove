## Description

When a task moves from "doing" to "review", the execution session (tmux session containing the execution/agent terminal) is being hidden from view. The UI in `TaskDetailPanel.tsx:647-649` sets `sessionMode` to "plan" for non-"doing" statuses, which causes it to look for `terminalPlanSession` instead of `terminalExecSession`. Since tasks in review have their execution session stored in `terminalExecSession`, the terminal becomes hidden from view.

The tmux session is NOT being destroyed - it's still running in the background. The user wants to see the exec terminal in the review column so they can iterate on the execution.

**Out of scope:**

- Moving to "done" status (different workflow, may archive or handle differently)
- Handling tasks in review that have no `terminalExecSession` (edge case, show empty state gracefully)

## Definition of Done

- [x] Change `TaskDetailPanel.tsx` line 649 to show exec terminal for both "doing" AND "review" statuses: `(task.status === "doing" || task.status === "review") ? "exec" : "plan"`
- [ ] Verify that tasks in review column show the execution terminal with the full chat history
- [ ] Test moving from review back to doing preserves the exec terminal behavior
