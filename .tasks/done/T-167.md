## Description

Intermittent task opening failures caused by React Hooks rule violations. The app crashes with "Cannot read properties of undefined (reading 'length')" error in useEffect and "Hooks called unconditionally" errors.

### Root Causes

1. **TaskDetailPanel.tsx (lines 258-270)**: `updateBoardTab` from Zustand store changes identity between renders, causing `setActiveTab` useCallback to recreate, which changes the useEffect dependency array size. React detects this as unstable dependencies.

2. **TaskSwitcherModal.tsx (lines 298+)**: Missing `key` prop on list items rendered via `.map()`.

### Fixes Required

1. Fix unstable useEffect in TaskDetailPanel.tsx by using a ref to hold stable callback
2. Add missing key props to TaskSwitcherModal list items
