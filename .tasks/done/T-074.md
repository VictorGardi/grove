## Description

Replace the workspace `<select>` dropdown in `WorkspaceDefaultsForm` with a clickable card/box grid (matching the theme card pattern). Each workspace card displays:

- Workspace name (bold, truncated)
- Workspace path (muted, truncated with `~` home shorthand)
- Current git branch (already available in `WorkspaceInfo`)

When a workspace card is clicked, the agent/model settings form appears below the grid. The active workspace from the store should be pre-selected. When no workspace exists, show an empty state message.

### Implementation approach

1. Extract `AgentModelForm` subcomponent from `WorkspaceDefaultsForm` (accepts `workspacePath` + `isDisabled` props)
2. Create `WorkspaceCardGrid` component with:
   - Grid layout (3 columns, matching theme grid)
   - `<button>` elements with `aria-pressed` for accessibility
   - Checkmark badge on selected card
   - Disabled/grayed-out state for `exists: false` workspaces
3. Update `Settings.tsx` to compose the grid + form
4. Add CSS for workspace cards (reuse theme card patterns, avoid theme-specific class coupling)

### Edge cases to handle

- `exists: false` workspaces: show disabled card with visual indicator
- Empty workspace list: show "No workspaces configured" message
- Long paths: truncate with tooltip on hover
- Stale selection: if selected workspace is removed, reset selection gracefully
