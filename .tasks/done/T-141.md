## Description

**Root Cause:** The task switcher uses `useAllTasksStore` which only fetches tasks when a workspace is first loaded (line 54: `if (!allTasks.has(ws.path))`). Unlike the task detail panel which refreshes via `workspace:dataChanged` events, the task switcher never refreshes when task files change on disk.

**Fix:** Add an effect in App.tsx that listens to `workspace:dataChanged` and refreshes all workspaces in `useAllTasksStore`, mirroring the existing pattern for the task detail panel.

Implement in `src/renderer/src/App.tsx`:

```tsx
// Add after the existing useAllTasksStore effect (around line 58):
useEffect(() => {
  if (!window.api.data) return;
  const unsub = window.api.data.onChanged(async () => {
    // Re-fetch tasks for all workspaces to update task switcher
    for (const ws of workspaces) {
      await fetchTasksForWorkspace(ws.path);
    }
  });
  return unsub;
}, [workspaces, fetchTasksForWorkspace]);
```

This ensures the task switcher updates immediately when task files change (create/edit/move/delete), showing the correct status (backlog/doing/review/done) without requiring an app reload.
