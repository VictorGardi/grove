## Description

Optimize Grove's critical path performance to achieve "instant illusion" task switching and efficient terminal handling. Focus on existing operations only — no new features.

### Current Bottlenecks

1. **Task switching re-fetches from disk** (`switchToTask()` in `useTaskSwitcherStore.ts:243`):
   - Calls `window.api.data.fetch()` on every workspace switch
   - No caching of task metadata

2. **No task state preloading**:
   - Board/terminal state restored but task data re-fetched
   - Context recomputed on each switch

3. **Terminal handling**:
   - PTYs created/destroyed on each interaction
   - No virtualization for diff views
   - Tab filtering on every render without memoization

### Performance Strategy

- **Task switching**: Preload task state, keep sessions alive, swap visible context only
- **Terminal**: Treat tmux as source of truth, stream output efficiently
- **Caching**: Cache task metadata, recent tasks, workspace state
- **Avoid**: Disk reads on every switch, recomputing context
