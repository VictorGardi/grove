## Description

### Root Cause

On app reload (Cmd+R) while on a task:

1. localStorage restores `activeView: "task"` from browser storage
2. `restoreBoardState` tries to restore the saved task ID for the workspace
3. **If the task was deleted** (or doesn't exist in the current tasks), it does nothing
4. But `activeView` is still "task" → "No task selected" placeholder shows

The bug is in `restoreBoardState` (useWorkspaceStore.ts:178-191): it silently skips restoring if the task doesn't exist, but it doesn't correct the `activeView`.

### Fix

In `restoreBoardState`, when the saved task doesn't exist:

1. Still call `setSelectedTask(null)` to ensure clean state
2. **Also redirect to "home" view** by calling `useNavStore.getState().setActiveView("home")`

This handles:

- Task was deleted before reload
- Task moved to different workspace
- Any case where saved task ID is stale

- [x] Same behavior if task was deleted before reload
