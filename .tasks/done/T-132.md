## Description

Currently, Cmd+T shortcut only creates a new task when in the board view. Users expect Cmd+T to always create a new task, including when in the task-centric view introduced in T-130.

The fix is simple: remove the `if (nav.activeView === "board")` condition from the Cmd+T handler in `useKeyboardShortcuts.ts` so it works in all views.

**Code change location:** `src/renderer/src/hooks/useKeyboardShortcuts.ts`, lines 164-172

Current code:

```typescript
// Cmd+T: create new task — works from board view
if (mod && (e.key === "t" || e.key === "T")) {
  const nav = useNavStore.getState();
  if (nav.activeView === "board") {
    e.preventDefault();
    createTask("New task");
    return;
  }
}
```

New code (remove the view check):

```typescript
// Cmd+T: create new task
if (mod && (e.key === "t" || e.key === "T")) {
  e.preventDefault();
  createTask("New task");
  return;
}
```
