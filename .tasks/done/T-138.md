## Description

When opening a task from the command palette (Cmd+Shift+K), a terminal panel unexpectedly opens for tasks that have an active terminal session (isRunning = true). The expected behavior is that NO terminal should open when opening a task from the command palette.

**Root cause**: In `src/renderer/src/stores/useTaskSwitcherStore.ts` at lines 214-216, the `switchToTask` function unconditionally opens the terminal panel for running tasks:

```typescript
if (sortedTask.isRunning) {
  navStore.setTerminalPanelOpen(true);
}
```

**Fix**: Remove these lines (214-216) to prevent automatic terminal opening when switching to a task via the task switcher.

## Definition of Done

- [x] Terminal does NOT open when opening a task from Cmd+Shift+K
- [x] Terminal does NOT open when switching to a task via Cmd+Shift+Tab (toggle last task)
