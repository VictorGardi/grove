## Description

**Feature 1: Workspace Filtering (Right-click)**

- Right-click a workspace in sidebar to show/hide tasks
- Context menu option: "Hide Tasks" or "Show Tasks"
- Hidden workspaces shown grayed out in sidebar

**Feature 2: Task Switcher Quick Navigation**

- Task switcher (cmd+k / ctrl+k) should include hidden pages (Settings, Agents) that appear when search query matches
- Matching is prefix-based: "Set" matches "Settings", "Agen" matches "Agents" (case-insensitive)
- Results only appear when the search query is a prefix match
- Default state: Settings and Agents are NOT visible in task switcher results

## Definition of Done

- [x] Right-click context menu to hide/show tasks on workspace
- [x] Task list views (backlog/doing/review/done) exclude tasks from hidden workspaces
- [x] Task switcher excludes hidden workspace tasks but shows Settings/Agents on prefix match
- [x] Settings page appears in task switcher when search query starts with "Set" (case-insensitive)
- [x] Agents page appears in task switcher when search query starts with "Agen" (case-insensitive)
- [x] Hidden workspaces shown grayed out in sidebar
- [x] Persist hidden state across app restarts

## Notes

Changed approach from Settings UI to right-click context menu. Hidden state is now a simple boolean (no timestamp-based).
