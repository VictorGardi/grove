## Description

Add support for rendering mermaid diagrams in markdown files displayed in the file viewer.

Currently, `MarkdownViewer.tsx` uses Shiki to syntax-highlight all code blocks, including `mermaid` blocks. This enhancement will detect mermaid code blocks and render them as actual SVG diagrams using the mermaid library.

### Technical Context

- Markdown files are rendered via `react-markdown` in `src/renderer/src/components/Files/MarkdownViewer.tsx`
- Code blocks are intercepted in the `pre` component override and routed to `CodeBlock`
- Grove uses themes: `grove-dark`, `mocha`, `latte` (stored in `useThemeStore`)
- No existing dynamic import patterns in the codebase

## Definition of Done

- [x] Mermaid package added as dependency
- [x] `CodeBlock` detects `lang === "mermaid"` and renders via mermaid instead of Shiki
- [x] Lazy-load mermaid to avoid blocking initial render
- [x] Valid mermaid syntax renders as an SVG diagram
- [x] Invalid mermaid syntax shows error message inline (does not crash)
- [x] Mermaid container styled consistently with existing code block appearance
- [x] Multiple mermaid blocks in one file work without ID collisions
- [x] Mermaid diagrams sized appropriately (scrollable if too large)
- [ ] Theme synchronization (Phase 2 - deferred to separate task)

## Implementation Plan

### Phase 1: Core Rendering (MVP)

1. **Add mermaid dependency**
   - Install `mermaid` npm package (specify version ~11.x for stable API)

2. **Create MermaidDiagram component** (`src/renderer/src/components/Files/MermaidDiagram.tsx`)
   - Dynamic import of mermaid library
   - Use `useEffect` + `mermaid.run()` to render after mount
   - Use ref to target container div for mermaid injection
   - Generate unique ID per instance to avoid collisions: `mermaid-${instanceId}`
   - Wrap in try/catch to handle parse errors gracefully

3. **Modify `CodeBlock` in `MarkdownViewer.tsx`**
   - Add check: `if (lang === "mermaid") return <MermaidDiagram code={code} />`
   - Keep existing Shiki path for all other languages

4. **Error handling UX**
   - On parse error: render container with error icon + error message
   - Style error state consistently with code block appearance

5. **Container styling**
   - `max-width: 100%`, `overflow-x: auto` for wide diagrams
   - `max-height: 500px`, `overflow-y: auto` for tall diagrams
   - Center the SVG

### Phase 2: Theme Synchronization (Deferred)

Research needed before implementation:

- Map Grove themes to mermaid theme names
- Subscribe to `useThemeStore` changes
- Call `mermaid.initialize({ theme: ... })` on theme change
- Re-render all visible mermaid diagrams

### Edge Cases to Handle

| Edge Case               | Solution                            |
| ----------------------- | ----------------------------------- |
| Multiple mermaid blocks | Each `MermaidDiagram` has unique ID |
| Very large diagrams     | Container with overflow scroll      |
| Invalid syntax          | Catch error, show inline message    |
| Rapid file navigation   | Lazy load + cleanup on unmount      |
| Same file opened twice  | Unique IDs prevent collisions       |
