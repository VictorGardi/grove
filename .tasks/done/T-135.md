## Description

Two issues with Cmd+Shift+K (task switcher):

### Issue 1: Done tasks show at top

The task switcher shows ALL tasks including those with status "done". The sorting is based on:

- Running tasks (+10000 points)
- Last viewed timestamp
- Created timestamp (for tasks never viewed)

Old "done" tasks incorrectly appear at the top because they have older `created` timestamps. The fix requires:

1. Change the default filter in `useTaskSwitcherStore.ts` `getSortedTasks` to EXCLUDE tasks with status "done"
2. Add a toggle/filter control in the TaskSwitcherModal UI to optionally include done tasks

### Issue 2: Task titles not showing correctly

Some tasks show "New Task" or incorrect titles (e.g., "T 131") in the switcher. Root causes:

1. Tasks created without a proper title have `title: New task` in frontmatter
2. Some task files have malformed YAML (multiple frontmatter blocks), causing parsing issues
3. The filename-based fallback in `parseTaskFile` isn't being used effectively

Fix requires:

1. Improve title parsing to use filename fallback when title is missing or generic "New task"
2. Fix malformed task files that have duplicate frontmatter blocks
