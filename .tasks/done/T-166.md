## Description

**Root Causes**:

1. `includeDoneTasks` defaults to `true` at `useTaskSwitcherStore.ts:56`, including done tasks in the sorting pool
2. The filter at lines 214-216 shows tasks if they're running OR recently viewed - so a done task (T-163) that was recently viewed appears

**Proposed Fix**:

1. Change `includeDoneTasks` default from `true` to `false`
2. Modify the "no search query" filter (lines 214-216) to exclude tasks with status "done" entirely - even if recently viewed
3. Wire up the existing `toggleIncludeDoneTasks()` function to a keyboard shortcut (e.g., `Cmd+Shift+D`)
4. Add a visual indicator/hint in the task switcher UI showing the current state
