## Description

The chat shows "Agent exited with code 1. Is opencode installed and on PATH? If so, try New session." for a few seconds after each user message, then disappears once the agent starts responding.

### Root Cause Analysis

The exit warning appears when:

```typescript
const showExitWarning =
  lastMsg?.role === "agent" &&
  !lastMsg.isStreaming &&
  !isRunning &&
  ((lastExitCode != null && lastExitCode !== 0) ||
    (lastMsgHasNoOutput && messages.length > 0));
```

**Issue**: `lastExitCode` is preserved from previous sessions/runs. When `startAgentMessage()` is called for a new message, it clears `lastStderr` but NOT `lastExitCode`. If the previous session had `lastExitCode: 1` (from a crash), and there's any gap where `isStreaming: false` and `isRunning: false`, the stale exit code triggers the warning.

The warning should only appear for exit codes from the **current session**, not stale ones from previous sessions.

### Chosen Solution: Option A + C

Selected after code review. Simpler than Option B and addresses both the root cause (stale exit codes) and UX gap (thinking indicator).

#### Option A: Clear `lastExitCode` when starting new message

- In `startAgentMessage()` in `usePlanStore.ts`, also set `lastExitCode: null`
- Simple fix - new messages start fresh
- Con: Hides crash info from previous runs (but this is acceptable since user already saw the warning for those)

#### Option B: Track which turn produced the exit code

- Add `turnId` counter incremented each time `startAgentMessage()` is called
- Store `exitCodeTurnId` alongside `lastExitCode`
- Only show warning if `exitCodeTurnId === lastMsg.turnId`
- Con: More complex, requires schema change

#### Option C: Add "Thinking..." indicator

- While waiting for first response (after `startAgentMessage()` but before first chunk), show a "Thinking..." or spinning indicator
- Addresses UX gap during startup delay
- Can be combined with either A or B

### Implementation Plan

1. **Clear `lastExitCode` in `startAgentMessage()`** - Add `lastExitCode: null` to state reset in `usePlanStore.ts:149`. This ensures new messages don't inherit stale exit codes from previous sessions.

2. **Add "Thinking..." indicator** - Show during startup delay instead of blank/error space
   - In `ChatMessage` component (`PlanChat.tsx:52`), when `msg.isStreaming && !msg.text && (!msg.content || msg.content.length === 0)`, display "Thinking..."
   - Use existing `thinkingBlock` style from `PlanChat.module.css:328`
   - Note: Check for empty array, not just falsy `content`
