## Description

The task switcher search (Cmd+K) only searches through active tasks. It excludes done tasks from search results, making it impossible to find completed tasks via search.

### Root Cause

In `useTaskSwitcherStore.ts`, done tasks are filtered out by default at line 159-161:

```typescript
if (task.status === "done" && !state.includeDoneTasks) {
  continue;
}
```

The default value of `includeDoneTasks` is `false` (line 56).

### Solution

Change the default value of `includeDoneTasks` from `false` to `true` in the store's initial state. This will include done tasks in search results while preserving the existing toggle functionality for users who want to hide done tasks.

### Files to Modify

- `src/renderer/src/stores/useTaskSwitcherStore.ts` - Line 56: change `includeDoneTasks: false` to `includeDoneTasks: true`
